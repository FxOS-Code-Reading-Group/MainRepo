00009 /*
00010 ** API for NSPR threads. On some architectures (Mac OS Classic
00011 ** notably) pre-emptibility is not guaranteed. Hard priority scheduling
00012 ** is not guaranteed, so programming using priority based synchronization
00013 ** is a no-no.
00014 **
00015 ** NSPR threads are scheduled based loosely on their client set priority.
00016 ** In general, a thread of a higher priority has a statistically better
00017 ** chance of running relative to threads of lower priority. However,
00018 ** NSPR uses multiple strategies to provide execution vehicles for thread
00019 ** abstraction of various host platforms. As it turns out, there is little
00020 ** NSPR can do to affect the scheduling attributes of "GLOBAL" threads.
00021 ** However, a semblance of GLOBAL threads is used to implement "LOCAL"
00022 ** threads. An arbitrary number of such LOCAL threads can be assigned to
00023 ** a single GLOBAL thread.
00024 **
00025 ** For scheduling, NSPR will attempt to run the highest priority LOCAL
00026 ** thread associated with a given GLOBAL thread. It is further assumed
00027 ** that the host OS will apply some form of "fair" scheduling on the
00028 ** GLOBAL threads.
00029 **
00030 ** Threads have a "system flag" which when set indicates the thread
00031 ** doesn't count for determining when the process should exit (the
00032 ** process exits when the last user thread exits).
00033 **
00034 ** Threads also have a "scope flag" which controls whether the threads
00035 ** are scheduled in the local scope or scheduled by the OS globally. This 
00036 ** indicates whether a thread is permanently bound to a native OS thread. 
00037 ** An unbound thread competes for scheduling resources in the same process.
00038 **
00039 ** Another flag is "state flag" which control whether the thread is joinable.
00040 ** It allows other threads to wait for the created thread to reach completion.
00041 **
00042 ** Threads can have "per-thread-data" attached to them. Each thread has a
00043 ** per-thread error number and error string which are updated when NSPR
00044 ** operations fail.
00045 */

00080 /*
00081 ** Create a new thread:
00082 **     "type" is the type of thread to create
00083 **     "start(arg)" will be invoked as the threads "main"
00084 **     "priority" will be created thread's priority
00085 **     "scope" will specify whether the thread is local or global
00086 **     "state" will specify whether the thread is joinable or not
00087 **     "stackSize" the size of the stack, in bytes. The value can be zero
00088 **        and then a machine specific stack size will be chosen.
00089 **
00090 ** This can return NULL if some kind of error occurs, such as if memory is
00091 ** tight.
00092 **
00093 ** If you want the thread to start up waiting for the creator to do
00094 ** something, enter a lock before creating the thread and then have the
00095 ** threads start routine enter and exit the same lock. When you are ready
00096 ** for the thread to run, exit the lock.
00097 **
00098 ** If you want to detect the completion of the created thread, the thread
00099 ** should be created joinable.  Then, use PR_JoinThread to synchrnoize the
00100 ** termination of another thread.
00101 **
00102 ** When the start function returns the thread exits. If it is the last
00103 ** PR_USER_THREAD to exit then the process exits.
00104 */
00105 NSPR_API(PRThread*) PR_CreateThread(PRThreadType type,
00106                      void (PR_CALLBACK *start)(void *arg),
00107                      void *arg,
00108                      PRThreadPriority priority,
00109                      PRThreadScope scope,
00110                      PRThreadState state,
00111                      PRUint32 stackSize);

00113 /*
00114 ** Wait for thread termination:
00115 **     "thread" is the target thread 
00116 **
00117 ** This can return PR_FAILURE if no joinable thread could be found 
00118 ** corresponding to the specified target thread.
00119 **
00120 ** The calling thread is blocked until the target thread completes.
00121 ** Several threads cannot wait for the same thread to complete; one thread

00165 /*
00166 ** This routine returns a new index for per-thread-private data table. 
00167 ** The index is visible to all threads within a process. This index can 
00168 ** be used with the PR_SetThreadPrivate() and PR_GetThreadPrivate() routines 
00169 ** to save and retrieve data associated with the index for a thread.
00170 **
00171 ** Each index is associationed with a destructor function ('dtor'). The function
00172 ** may be specified as NULL when the index is created. If it is not NULL, the
00173 ** function will be called when:
00174 **      - the thread exits and the private data for the associated index

00180 ** A thread can only get access to its own thread-specific-data.
00181 **

00183 ** is NULL, and upon thread creation the value associated with all indices for 
00184 ** that thread is NULL.

00187 ** allowed.
