00033 // Declaration objects have unusual lifetime rules.  Every declaration
00034 // begins life in an invalid state which ends when InitializeEmpty or
00035 // CompressFrom is called upon it.  After that, it can be attached to
00036 // exactly one style rule, and will be destroyed when that style rule
00037 // is destroyed.  A declaration becomes immutable when its style rule's
00038 // |RuleMatched| method is called; after that, it must be copied before
00039 // it can be modified, which is taken care of by |EnsureMutable|.
