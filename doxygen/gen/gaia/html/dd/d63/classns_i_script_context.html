virtual nsresult nsIScriptContext::BindCompiledEventHandler ( nsISupports *  aTarget,
JS::Handle< JSObject * >  aScope,
JS::Handle< JSObject * >  aHandler,
JS::MutableHandle< JSObject * >  aBoundHandler 
) [pure virtual]

Bind an already-compiled event handler function to the given target. Scripting languages with static scoping must re-bind the scope chain for aHandler to begin (after the activation scope for aHandler itself, typically) with aTarget's scope.

The result of the bind operation is a new handler object, with principals now set and scope set as above. This is returned in aBoundHandler. When this function is called, aBoundHandler is expected to not be holding an object.

引数:
aTargetan object telling the scope in which to bind the compiled event handler function. The context will presumably associate this nsISupports with a native script object.
aScopethe scope in which the script object for aTarget should be looked for.
aHandlerthe function object to bind, created by an earlier call to CompileEventHandler
aBoundHandler[out] the result of the bind operation.
戻り値:
NS_OK if the function was successfully bound

Compile and execute a script.

引数:
aScripta string representing the script to be executed
aScopeObjecta script object for the scope to execute in.
aOptionsan options object. You probably want to at least set filename and line number. The principal is computed internally, though 'originPrincipals' may be passed.
aCoerceToStringif the return value is not JSVAL_VOID, convert it to a string before returning.
aRetValuethe result of executing the script. Pass null if you don't care about the result. Note that asking for a result will deoptimize your script somewhat in many cases.
