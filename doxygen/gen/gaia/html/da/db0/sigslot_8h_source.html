00009 //      
00010 //              (see also the full documentation at http://sigslot.sourceforge.net/)
00011 //
00012 //      #define switches
00013 //          SIGSLOT_PURE_ISO            - Define this to force ISO C++ compliance. This also disables
00014 //                                        all of the thread safety support on platforms where it is 
00015 //                                        available.
00016 //
00017 //          SIGSLOT_USE_POSIX_THREADS   - Force use of Posix threads when using a C++ compiler other than
00018 //                                        gcc on a platform that supports Posix threads. (When using gcc,
00019 //                                        this is the default - use SIGSLOT_PURE_ISO to disable this if 
00020 //                                        necessary)
00021 //
00022 //          SIGSLOT_DEFAULT_MT_POLICY   - Where thread support is enabled, this defaults to multi_threaded_global.
00023 //                                        Otherwise, the default is single_threaded. #define this yourself to
00024 //                                        override the default. In pure ISO mode, anything other than
00025 //                                        single_threaded will cause a compiler error.
00026 //
00027 //      PLATFORM NOTES
00028 //
00029 //          Win32                       - On Win32, the WIN32 symbol must be #defined. Most mainstream
00030 //                                        compilers do this by default, but you may need to define it
00031 //                                        yourself if your build environment is less standard. This causes
00032 //                                        the Win32 thread support to be compiled in and used automatically.
00033 //
00034 //          Unix/Linux/BSD, etc.        - If you're using gcc, it is assumed that you have Posix threads
00035 //                                        available, so they are used automatically. You can override this
00036 //                                        (as under Windows) with the SIGSLOT_PURE_ISO switch. If you're using
00037 //                                        something other than gcc but still want to use Posix threads, you
00038 //                                        need to #define SIGSLOT_USE_POSIX_THREADS.
00039 //
00040 //          ISO C++                     - If none of the supported platforms are detected, or if
00041 //                                        SIGSLOT_PURE_ISO is defined, all multithreading support is turned off,
00042 //                                        along with any code that might cause a pure ISO C++ environment to
00043 //                                        complain. Before you ask, gcc -ansi -pedantic won't compile this 
00044 //                                        library, but gcc -ansi is fine. Pedantic mode seems to throw a lot of
00045 //                                        errors that aren't really there. If you feel like investigating this,
00046 //                                        please contact the author.
00047 //
00048 //      
00049 //      THREADING MODES
00050 //
00051 //          single_threaded             - Your program is assumed to be single threaded from the point of view
00052 //                                        of signal/slot usage (i.e. all objects using signals and slots are
00053 //                                        created and destroyed from a single thread). Behaviour if objects are
00054 //                                        destroyed concurrently is undefined (i.e. you'll get the occasional
00055 //                                        segmentation fault/memory exception).
00056 //
00057 //          multi_threaded_global       - Your program is assumed to be multi threaded. Objects using signals and
00058 //                                        slots can be safely created and destroyed from any thread, even when
00059 //                                        connections exist. In multi_threaded_global mode, this is achieved by a
00060 //                                        single global mutex (actually a critical section on Windows because they
00061 //                                        are faster). This option uses less OS resources, but results in more
00062 //                                        opportunities for contention, possibly resulting in more context switches
00063 //                                        than are strictly necessary.
00064 //
00065 //          multi_threaded_local        - Behaviour in this mode is essentially the same as multi_threaded_global,
00066 //                                        except that each signal, and each object that inherits has_slots, all 
00067 //                                        have their own mutex/critical section. In practice, this means that
00068 //                                        mutex collisions (and hence context switches) only happen if they are
00069 //                                        absolutely essential. However, on some platforms, creating a lot of 
00070 //                                        mutexes can slow down the whole OS, so use this option with care.
00071 //
00072 //      USING THE LIBRARY
00073 //
00074 //          See the full documentation at http://sigslot.sourceforge.net/
00075 //
00076 //
00077 // Libjingle specific:
00078 // This file has been modified such that has_slots and signalx do not have to be
00079 // using the same threading requirements. E.g. it is possible to connect a
00080 // has_slots<single_threaded> and signal0<multi_threaded_local> or
00081 // has_slots<multi_threaded_local> and signal0<single_threaded>.

00083 // to connect or disconnect to signalx concurrently or data race may occur.
00084 // If signalx is single threaded the user must ensure that disconnect, connect
00085 // or signal is not happening concurrently or data race may occur.
