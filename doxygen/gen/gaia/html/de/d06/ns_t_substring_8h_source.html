00815       // Some terminology:
00816       //
00817       //   "dependent buffer"    A dependent buffer is one that the string class
00818       //                         does not own.  The string class relies on some
00819       //                         external code to ensure the lifetime of the
00820       //                         dependent buffer.
00821       //
00822       //   "shared buffer"       A shared buffer is one that the string class
00823       //                         allocates.  When it allocates a shared string
00824       //                         buffer, it allocates some additional space at
00825       //                         the beginning of the buffer for additional 
00826       //                         fields, including a reference count and a 
00827       //                         buffer length.  See nsStringHeader.
00828       //                         
00829       //   "adopted buffer"      An adopted buffer is a raw string buffer
00830       //                         allocated on the heap (using nsMemory::Alloc)
00831       //                         of which the string class subsumes ownership.
00832       //
00833       // Some comments about the string flags:
00834       //
00835       //   F_SHARED, F_OWNED, and F_FIXED are all mutually exlusive.  They
00836       //   indicate the allocation type of mData.  If none of these flags
00837       //   are set, then the string buffer is dependent.
00838       //
00839       //   F_SHARED, F_OWNED, or F_FIXED imply F_TERMINATED.  This is because
00840       //   the string classes always allocate null-terminated buffers, and
00841       //   non-terminated substrings are always dependent.

00843       //   F_VOIDED implies F_TERMINATED, and moreover it implies that mData
00844       //   points to char_traits::sEmptyBuffer.  Therefore, F_VOIDED is
00845       //   mutually exclusive with F_SHARED, F_OWNED, and F_FIXED.
