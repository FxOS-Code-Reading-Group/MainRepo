00001 The intent is that this folder eventually becomes a stand alone lib.
00002 
00003 At the base level (provider)
00004 there are a number of content providers.
00005 
00006 Content providers each are responsible for providing calendars and
00007 events which we eventually consume/persist in the form of "models" which
00008 represent the local state of a resource. 
00009 
00010 The difference (and intent of separation between models and provider
00011 objects) is provider object always represent the state of the *server*
00012 where models may or may not be in the same state and may also
00013 contain changes which only we persist *locally*.
00014 
00015 Each provider object _must_ use the same API to provide
00016 an abstraction layer for models/stores. Each top level
00017 provider object (e.g. Local) contains a list of capabilities
00018 which effect how the models/stores interact with them.
