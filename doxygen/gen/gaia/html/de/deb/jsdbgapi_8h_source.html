00354 /*
00355  * This hook captures high level script execution and function calls (JS or
00356  * native).  It is used by JS_SetExecuteHook to hook top level scripts and by
00357  * JS_SetCallHook to hook function calls.  It will get called twice per script
00358  * or function call: just before execution begins and just after it finishes.
00359  * In both cases the 'current' frame is that of the executing code.
00360  *
00361  * The 'before' param is JS_TRUE for the hook invocation before the execution
00362  * and JS_FALSE for the invocation after the code has run.
00363  *
00364  * The 'ok' param is significant only on the post execution invocation to
00365  * signify whether or not the code completed 'normally'.
00366  *
00367  * The 'closure' param is as passed to JS_SetExecuteHook or JS_SetCallHook
00368  * for the 'before'invocation, but is whatever value is returned from that
00369  * invocation for the 'after' invocation. Thus, the hook implementor *could*
00370  * allocate a structure in the 'before' invocation and return a pointer to that
00371  * structure. The pointer would then be handed to the hook for the 'after'
00372  * invocation. Alternately, the 'before' could just return the same value as
00373  * in 'closure' to cause the 'after' invocation to be called with the same
00374  * 'closure' value as the 'before'.
00375  *
00376  * Returning NULL in the 'before' hook will cause the 'after' hook *not* to
00377  * be called.
00378  */
00379 typedef void *
00380 (* JSInterpreterHook)(JSContext *cx, JSAbstractFramePtr frame, bool isConstructing,
00381                       JSBool before, JSBool *ok, void *closure);
