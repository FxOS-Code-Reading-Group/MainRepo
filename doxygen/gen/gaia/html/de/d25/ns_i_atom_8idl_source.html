00081 /*
00082  * The three forms of NS_NewAtom and do_GetAtom (for use with
00083  * |nsCOMPtr<nsIAtom>|) return the atom for the string given.  At any
00084  * given time there will always be one atom representing a given string.
00085  * Atoms are intended to make string comparison cheaper by simplifying
00086  * it to pointer equality.  A pointer to the atom that does not own a
00087  * reference is not guaranteed to be valid.
00088  *
00089  * The three forms of NS_NewPermanentAtom and do_GetPermanentAtom return
00090  * the atom for the given string and ensure that the atom is permanent.
00091  * An atom that is permanent will exist (occupy space at a specific
00092  * location in memory) until XPCOM is shut down.  The advantage of
00093  * permanent atoms is that they do not need to maintain a reference
00094  * count, which requires locking and hurts performance.
00095  */
