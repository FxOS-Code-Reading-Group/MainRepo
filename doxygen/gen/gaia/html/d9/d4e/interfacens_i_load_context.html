説明

An nsILoadContext represents the context of a load. This interface can be queried for various information about where the load is happening.

readonly attribute unsigned long nsILoadContext::appId

Returns the app id of the app the load is occurring is in. Returns nsIScriptSecurityManager::NO_APP_ID if the load is not part of an app.

readonly attribute nsIDOMWindow nsILoadContext::associatedWindow

associatedWindow is the window with which the load is associated, if any. Note that the load may be triggered by a document which is different from the document in associatedWindow, and in fact the source of the load need not be same-origin with the document in associatedWindow. This attribute may be null if there is no associated window.

nsILoadContext.idl の 27 行で定義されています。

readonly attribute boolean nsILoadContext::isContent

True if the load context is content (as opposed to chrome). This is determined based on the type of window the load is performed in, NOT based on any URIs that might be around.

nsILoadContext.idl の 62 行で定義されています。

readonly attribute boolean nsILoadContext::isInBrowserElement

Returns true iff the load is occurring inside a browser element.

readonly attribute nsIDOMElement nsILoadContext::topFrameElement

topFrameElement is the <iframe> or <frame> element which contains the topWindow with which the load is associated.

Note that we may have a topFrameElement even when we don't have an associatedWindow, if the topFrameElement's content lives out of process.

nsILoadContext.idl の 44 行で定義されています。

readonly attribute nsIDOMWindow nsILoadContext::topWindow

topWindow is the top window which is of same type as associatedWindow. This is equivalent to associatedWindow.top, but is provided here as a convenience. All the same caveats as associatedWindow of apply, of course. This attribute may be null if there is no associated window.
