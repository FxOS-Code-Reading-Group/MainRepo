説明

A ReadbackSink receives a stream of updates to a rectangle of pixels. These update callbacks are always called on the main thread, either during EndTransaction or from the event loop.

uint64_t  aSequenceNumber 
) [pure virtual]

Called by the layer system to indicate that the contents of part of the readback area are changing.

引数:
aRectis the rectangle of content that is being updated, in the coordinate system of the ReadbackLayer.
aSequenceNumberupdates issued out of order should be ignored. Only use updates whose sequence counter is greater than all other updates seen so far. Return null when a non-fresh sequence value is given.
戻り値:
a context into which the update should be drawn. This should be set up to clip to aRect. Zero should never be passed as a sequence number. If this returns null, EndUpdate should NOT be called. If it returns non-null, EndUpdate must be called.

We don't support partially unknown backgrounds. Therefore, the first BeginUpdate after a SetUnknown will have the complete background.
