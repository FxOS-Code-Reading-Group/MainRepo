説明

A ReadbackLayer never renders anything. It enables clients to extract the rendered contents of the layer tree below the ReadbackLayer. The rendered contents are delivered asynchronously via calls to a ReadbackSink object supplied by the client.

This is a "best effort" API; it is possible for the layer system to tell the ReadbackSink that the contents of the readback area are unknown.

This API exists to work around the limitations of transparent windowless plugin rendering APIs. It should not be used for anything else.

void mozilla::layers::ReadbackLayer::SetSink ( ReadbackSink *  aSink)

CONSTRUCTION PHASE ONLY Set the callback object to which readback updates will be delivered. This also resets the "needed rectangle" so that on the next layer tree transaction we will try to deliver the full contents of the readback area to the sink. This layer takes ownership of the sink. It will be deleted when the layer is destroyed or when a new sink is set. Initially the contents of the readback area are completely unknown.
