00001 # Gaia [![Build Status](https://travis-ci.org/mozilla-b2g/gaia.png)](https://travis-ci.org/mozilla-b2g/gaia)

00027 ## Tests
00028 
00029 ### Unit Tests
00030 
00031 Unit tests for an app go in `apps/<APP>/test/unit/`.
00032 
00033 To run all the unit tests with B2G Desktop:
00034 
00035 1. Run `DEBUG=1 make`
00036 2. Run `make test-agent-server &`
00037 3. Run B2G Desktop and open the Test Agent app
00038 4. Run `make test-agent-test`
00039 
00040    or `make test-agent-test APP=<APP>` to run unit tests for a
00041    specific app
00042 
00043 More importantly, you can use test-agent-server to watch the files
00044 on the filesystem and execute relevant tests when they change:
00045 
00046 1. Run `DEBUG=1 make`
00047 2. Run `make test-agent-server &`
00048 3. Run B2G Desktop and open the Test Agent app
00049 4. Edit files and when you save them, glance at the console with
00050    test-agent-server running
00051 
00052 Note: If you add new files, you will need to restart test-agent-server.
00053 
00054 As a convenience, you can also use the `gaia-test` script to launch the
00055 test-agent-server and open the Test Agent app in firefox:
00056 
00057 1. Add firefox to your `$PATH` or set `$FIREFOX` to your preferred
00058    firefox/aurora/nightly binary.
00059 2. Run `./bin/gaia-test` to run the test-agent-server and launch firefox.
00060 3. Run `make test-agent-test` or modify files as described above.
00061 
00062 For more details on writing tests, see:
00063 https://developer.mozilla.org/en/Mozilla/Boot_to_Gecko/Gaia_Unit_Tests
00064 
00065 ### Integration Tests
00066 
00067 Gaia uses
00068 [marionette-js-runner](https://github.com/mozilla-b2g/marionette-js-runner)
00069 to run the tests with a custom builder for gaia. Tests should live with the rest of your apps code (in apps/my_app/test/marionette) and
00070 test files should end in _test.js.
00071 
00072 All integration tests run under a node environment. You need node >= 0.8
00073 for this to work predictably.
00074 
00075 Shared code for tests lives under shared/test/integration.
00076 
00077 #### Invoking a test
00078 
00079 ```sh
00080 ./bin/gaia-marionette <test> [test...]
00081 ```
00082 
00083 All options are passed to `./node_modules/.bin/marionette-mocha` so
00084 you can also use mocha commands like `--grep`, `--timeout` see `--help`
00085 for more options.
00086 
00087 #### Invoking all the tests
00088 
00089 NOTE: unless you tests end in _test.js they will not be
00090 automatically picked up by `make test-integration`.
00091 
00092 ```sh
00093 make test-integration
00094 ```

00104 #### Gotchas
00105 
00106 - For performance reasons we don't run `make profile` for each test
00107   run this means you need to manually remove the `profile-test`
00108   folder when you make changes to your apps.
00109 
00110 - If you don't have a b2g folder one will be downloaded for you.
00111   This can be problematic if you're offline. You can symlink a
00112   b2g-desktop directory to b2g/ in gaia to avoid the download.
00113 
00114 - If you have some weird node errors, try removing node_modules since
00115   things may be stale.
00116 
00117 - You get can get lots of debug information when running tests like
00118 this: `DEBUG=* ./bin/gaia-marionette name/of/test.js`
