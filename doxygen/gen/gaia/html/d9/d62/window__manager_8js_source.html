00004 //
00005 // This file calls getElementById without waiting for an onload event, so it
00006 // must have a defer attribute or be included at the end of the <body>.
00007 //
00008 // This module is responsible for launching apps and for allowing
00009 // the user to switch among apps and kill apps.  Specifically, it handles:
00010 //   launching apps,
00011 //   killing apps
00012 //   keeping track of the set of running apps (which we call tasks here)
00013 //   keeping track of which task is displayed (the foreground task)
00014 //   changing the foreground task
00015 //   hiding all apps to display the homescreen
00016 //   displaying the app switcher to allow the user to switch and kill apps
00017 //   performing appropriate transition animations between:
00018 //      the homescreen and an app
00019 //      the homescreen and the switcher
00020 //      an app and the homescreen
00021 //      the switcher and the homescreen
00022 //      the switcher and the current foreground task
00023 //      the switcher and a different task
00024 //   Handling Home key events to switch to the homescreen and the switcher
00025 //
00026 // The public API of the module is small. It defines an WindowManager object
00027 // with these methods:
00028 //
00029 //    launch(origin): switch to the specified running app
00030 //    kill(origin, callback): stop specified app
00031 //    reload(origin): reload the given app
00032 //    getDisplayedApp(): return the origin of the currently displayed app
00033 //    setOrientationForApp(origin): set the phone to orientation to a given app
00034 //    getAppFrame(origin): returns the iframe element for the specified origin
00035 //      which is assumed to be running.  This is only currently used
00036 //      for tests and chrome stuff: see the end of the file
00037 //    getRunningApps(): get the app references of the running apps.
00038 //
00039 // TODO
00040 // The "origin" does not actually refer to app's origin but rather a identifier
00041 // of the app reference that one gets from |getDisplayedApp()| or
00042 // iterates |getRunningApps|. The string is make up of the specified
00043 // launching entry point, origin, or the website url launched by wrapper.
00044 // It would be ideal if the variable get correctly named and it's rule is being
00045 // properly documented.
00046 // See https://bugzilla.mozilla.org/show_bug.cgi?id=796629
00047 //
