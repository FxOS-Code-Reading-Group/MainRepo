00239 /*
00240  * This interface is implemented by outside code and registered with xpconnect
00241  * via nsIXPConnect::setFunctionThisTranslator.
00242  *
00243  * The reason this exists is to support calls to JavaScript event callbacks
00244  * needed by the DOM via xpconnect from C++ code.
00245  *
00246  * We've added support for wrapping JS function objects as xpcom interfaces
00247  * by declaring the given interface as a [function] interface. However, to
00248  * support the requirements of JS event callbacks we need to call the JS
00249  * function with the 'this' set as the JSObject for which the event is being
00250  * fired; e.g. a form node.
00251  *
00252  * We've decided that for all cases we care about the appropriate 'this' object
00253  * can be derived from the first param in the call to the callback. In the
00254  * event handler case the first param is an event object.
00255  *
00256  * Though we can't change all the JS code so that it would setup its own 'this',
00257  * we can add plugin 'helper' support to xpconnect. And that is what we have
00258  * here.
00259  *
00260  * The idea is that at startup time some code that cares about this issue
00261  * (e.g. the DOM helper code) can register a nsIXPCFunctionThisTranslator
00262  * object with xpconnect to handle calls to [function] interfaces of a given
00263  * iid. When xpconnect goes to invoke a method on a wrapped JSObject for
00264  * an interface marked as [function], xpconnect will check if the first param
00265  * of the method is an xpcom object pointer and if so it will check to see if a
00266  * nsIXPCFunctionThisTranslator has been registered for the given iid of the
00267  * interface being called. If so it will call the translator and get an
00268  * interface pointer to use as the 'this' for the call. If the translator
00269  * returns a non-null interface pointer (which it should then have addref'd
00270  * since it is being returned as an out param), xpconnect will attempt to build
00271  * a wrapper around the pointer and get a JSObject from that wrapper to use
00272  * as the 'this' for the call.
00273  *
00274  * If a null interface pointer is returned then xpconnect will use the default
00275  * 'this' - the same JSObject as the function object it is calling.
00276  */
00277

00280 {
00281     nsISupports TranslateThis(in nsISupports aInitialThis);
00282 };
