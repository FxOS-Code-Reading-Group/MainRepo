列挙型

enum mozilla::a11y::roles::Role
列挙型の値:
NOTHING 

Used when accessible hans't strong defined role.

TITLEBAR 

Represents a title or caption bar for a window. It is used by MSAA only, supported automatically by MS Windows.

MENUBAR 

Represents the menu bar (positioned beneath the title bar of a window) from which menus are selected by the user. The role is used by xul:menubar or role="menubar".

SCROLLBAR 

Represents a vertical or horizontal scroll bar, which is part of the client area or used in a control.

GRIP 

Represents a special mouse pointer, which allows a user to manipulate user interface elements such as windows. For example, a user clicks and drags a sizing grip in the lower-right corner of a window to resize it.

SOUND 

Represents a system sound, which is associated with various system events.

CURSOR 

Represents the system mouse pointer.

CARET 

Represents the system caret. The role is supported for caret.

ALERT 

Represents an alert or a condition that a user should be notified about. Assistive Technologies typically respond to the role by reading the entire onscreen contents of containers advertising this role. Should be used for warning dialogs, etc. The role is used by xul:browsermessage, role="alert".

WINDOW 

Represents the window frame, which contains child objects such as a title bar, client, and other objects contained in a window. The role is supported automatically by MS Windows.

INTERNAL_FRAME 

A sub-document (<frame> or <iframe>)

MENUPOPUP 

Represents a menu, which presents a list of options from which the user can make a selection to perform an action. It is used for role="menu".

MENUITEM 

Represents a menu item, which is an entry in a menu that a user can choose to carry out a command, select an option. It is used for xul:menuitem, role="menuitem".

TOOLTIP 

Represents a ToolTip that provides helpful hints.

APPLICATION 

Represents a main window for an application. It is used for role="application". Also refer to APP_ROOT

DOCUMENT 

Represents a document window. A document window is always contained within an application window. It is used for role="document".

PANE 

Represents a pane within a frame or document window. Users can navigate between panes and within the contents of the current pane, but cannot navigate between items in different panes. Thus, panes represent a level of grouping lower than frame windows or documents, but above individual controls. It is used for the first child of a <frame> or <iframe>.

CHART 

Represents a graphical image used to represent data.

DIALOG 

Represents a dialog box or message box. It is used for xul:dialog, role="dialog".

BORDER 

Represents a window border.

GROUPING 

Logically groups other objects. There is not always a parent-child relationship between the grouping object and the objects it contains. It is used for html:textfield, xul:groupbox, role="group".

SEPARATOR 

Used to visually divide a space into two regions, such as a separator menu item or a bar that divides split panes within a window. It is used for xul:separator, html:hr, role="separator".

TOOLBAR 

Represents a toolbar, which is a grouping of controls (push buttons or toggle buttons) that provides easy access to frequently used features. It is used for xul:toolbar, role="toolbar".

STATUSBAR 

Represents a status bar, which is an area at the bottom of a window that displays information about the current operation, state of the application, or selected object. The status bar has multiple fields, which display different kinds of information. It is used for xul:statusbar.

TABLE 

Represents a table that contains rows and columns of cells, and optionally, row headers and column headers. It is used for html:table, role="grid". Also refer to the following role: COLUMNHEADER, ROWHEADER, COLUMN, ROW, CELL.

COLUMNHEADER 

Represents a column header, providing a visual label for a column in a table. It is used for XUL tree column headers, html:th, role="colheader". Also refer to TABLE.

ROWHEADER 

Represents a row header, which provides a visual label for a table row. It is used for role="rowheader". Also, see TABLE.

COLUMN 

Represents a column of cells within a table. Also, see TABLE.

ROW 

Represents a row of cells within a table. Also, see TABLE.

CELL 

Represents a cell within a table. It is used for html:td, xul:tree cell and xul:listcell. Also, see TABLE.

LINK 
HELPBALLOON 

Displays a Help topic in the form of a ToolTip or Help balloon.

CHARACTER 

Represents a cartoon-like graphic object, such as Microsoft Office Assistant, which is displayed to provide help to users of an application.

LIST 

Represents a list box, allowing the user to select one or more items. It is used for xul:listbox, html:select, role="list". See also LIST_ITEM.

LISTITEM 

Represents an item in a list. See also LIST.

OUTLINE 

Represents an outline or tree structure, such as a tree view control, that displays a hierarchical list and allows the user to expand and collapse branches. Is is used for role="tree".

OUTLINEITEM 

Represents an item in an outline or tree structure. It is used for role="treeitem".

PAGETAB 

Represents a page tab, it is a child of a page tab list. It is used for xul:tab, role="treeitem". Also refer to PAGETABLIST.

PROPERTYPAGE 

Represents a property sheet. It is used for xul:tabpanel, role="tabpanel".

INDICATOR 

Represents an indicator, such as a pointer graphic, that points to the current item.

GRAPHIC 

Represents a picture. Is is used for xul:image, html:img.

STATICTEXT 

Represents read-only text, such as labels for other controls or instructions in a dialog box. Static text cannot be modified or selected. Is is used for xul:label, xul:description, html:label, role="label".

TEXT_LEAF 

Represents selectable text that allows edits or is designated read-only.

PUSHBUTTON 

Represents a push button control. It is used for xul:button, html:button, role="button".

CHECKBUTTON 

Represents a check box control. It is used for xul:checkbox, html:input="checkbox", role="checkbox".

RADIOBUTTON 

Represents an option button, also called a radio button. It is one of a group of mutually exclusive options. All objects sharing a single parent that have this attribute are assumed to be part of single mutually exclusive group. It is used for xul:radio, html:input="radio", role="radio".

COMBOBOX 

Represents a combo box; an edit control with an associated list box that provides a set of predefined choices. It is used for html:select, xul:menulist, role="combobox".

DROPLIST 

Represents the calendar control.

PROGRESSBAR 

Represents a progress bar, dynamically showing the user the percent complete of an operation in progress. It is used for xul:progressmeter, role="progressbar".

DIAL 

Represents a dial or knob whose purpose is to allow a user to set a value.

HOTKEYFIELD 

Represents a hot-key field that allows the user to enter a combination or sequence of keystrokes.

SLIDER 

Represents a slider, which allows the user to adjust a setting in given increments between minimum and maximum values. It is used by xul:scale, role="slider".

SPINBUTTON 

Represents a spin box, which is a control that allows the user to increment or decrement the value displayed in a separate "buddy" control associated with the spin box. It is used for xul:spinbuttons.

DIAGRAM 

Represents a graphical image used to diagram data. It is used for svg:svg.

ANIMATION 

Represents an animation control, which contains content that changes over time, such as a control that displays a series of bitmap frames.

EQUATION 

Represents a mathematical equation. It is used by MATHML, where there is a rich DOM subtree for an equation. Use FLAT_EQUATION for

BUTTONDROPDOWN 

Represents a button that drops down a list of items.

BUTTONMENU 

Represents a button that drops down a menu.

BUTTONDROPDOWNGRID 

Represents a button that drops down a grid. It is used for xul:colorpicker.

WHITESPACE 

Represents blank space between other objects.

PAGETABLIST 

Represents a container of page tab controls. Is it used for xul:tabs, DHTML: role="tabs". Also refer to PAGETAB.

CLOCK 

Represents a control that displays time.

SPLITBUTTON 

Represents a button on a toolbar that has a drop-down list icon directly adjacent to the button.

IPADDRESS 

Represents an edit control designed for an Internet Protocol (IP) address. The edit control is divided into sections for the different parts of the IP address.

ACCEL_LABEL 

Represents a label control that has an accelerator.

ARROW 

Represents an arrow in one of the four cardinal directions.

CANVAS 

Represents a control that can be drawn into and is used to trap events. It is used for html:canvas.

CHECK_MENU_ITEM 

Represents a menu item with a check box.

COLOR_CHOOSER 

Represents a specialized dialog that lets the user choose a color.

DATE_EDITOR 

Represents control whose purpose is to allow a user to edit a date.

DESKTOP_ICON 

An iconified internal frame in an DESKTOP_PANE. Also refer to INTERNAL_FRAME.

DESKTOP_FRAME 

A desktop pane. A pane that supports internal frames and iconified versions of those internal frames.

DIRECTORY_PANE 

A directory pane. A pane that allows the user to navigate through and select the contents of a directory. May be used by a file chooser. Also refer to FILE_CHOOSER.

FILE_CHOOSER 

A file chooser. A specialized dialog that displays the files in the directory and lets the user select a file, browse a different directory, or specify a filename. May use the directory pane to show the contents of a directory. Also refer to DIRECTORY_PANE.

FONT_CHOOSER 

A font chooser. A font chooser is a component that lets the user pick various attributes for fonts.

CHROME_WINDOW 

Frame role. A top level window with a title bar, border, menu bar, etc. It is often used as the primary window for an application.

GLASS_PANE 

A glass pane. A pane that is guaranteed to be painted on top of all panes beneath it. Also refer to ROOT_PANE.

HTML_CONTAINER 

A document container for HTML, whose children represent the document content.

ICON 

A small fixed size picture, typically used to decorate components.

LABEL 

Presents an icon or short string in an interface.

LAYERED_PANE 

A layered pane. A specialized pane that allows its children to be drawn in layers, providing a form of stacking order. This is usually the pane that holds the menu bar as well as the pane that contains most of the visual components in a window. Also refer to GLASS_PANE and ROOT_PANE.

OPTION_PANE 

A specialized pane whose primary use is inside a dialog.

PASSWORD_TEXT 

A text object uses for passwords, or other places where the text content is not shown visibly to the user.

POPUP_MENU 

A temporary window that is usually used to offer the user a list of choices, and then hides when the user selects one of those choices.

RADIO_MENU_ITEM 

A radio button that is a menu item.

ROOT_PANE 

A root pane. A specialized pane that has a glass pane and a layered pane as its children. Also refer to GLASS_PANE and LAYERED_PANE.

SCROLL_PANE 

A scroll pane. An object that allows a user to incrementally view a large amount of information. Its children can include scroll bars and a viewport. Also refer to VIEW_PORT.

SPLIT_PANE 

A split pane. A specialized panel that presents two other panels at the same time. Between the two panels is a divider the user can manipulate to make one panel larger and the other panel smaller.

TABLE_COLUMN_HEADER 

The header for a column of a table. XXX: it looks this role is dupe of COLUMNHEADER.

TABLE_ROW_HEADER 

The header for a row of a table. XXX: it looks this role is dupe of ROWHEADER

TEAR_OFF_MENU_ITEM 

A menu item used to tear off and reattach its menu.

TERMINAL 

Represents an accessible terminal.

TEXT_CONTAINER 

Collection of objects that constitute a logical text entity.

TOGGLE_BUTTON 

A toggle button. A specialized push button that can be checked or unchecked, but does not provide a separate indicator for the current state.

TREE_TABLE 

Represent a control that is capable of expanding and collapsing rows as well as showing multiple columns of data.

VIEWPORT 

A viewport. An object usually used in a scroll pane. It represents the portion of the entire data that the user can see. As the user manipulates the scroll bars, the contents of the viewport can change. Also refer to SCROLL_PANE.

HEADER 

Header of a document page. Also refer to FOOTER.

FOOTER 

Footer of a document page. Also refer to HEADER.

PARAGRAPH 

A paragraph of text.

RULER 

A ruler such as those used in word processors.

AUTOCOMPLETE 

A text entry having dialog or list containing items for insertion into an entry widget, for instance a list of words for completion of a text entry. It is used for xul:textbox

EDITBAR 

An editable text object in a toolbar.

ENTRY 

An control whose textual content may be entered or modified by the user.

CAPTION 

A caption describing another object.

DOCUMENT_FRAME 

A visual frame or container which contains a view of document content. Document frames may occur within another Document instance, in which case the second document may be said to be embedded in the containing instance. HTML frames are often DOCUMENT_FRAME. Either this object, or a singleton descendant, should implement the Document interface.

HEADING 

Heading.

PAGE 

An object representing a page of document content. It is used in documents which are accessed by the user on a page by page basis.

SECTION 

A container of document content. An example of the use of this role is to represent an html:div.

REDUNDANT_OBJECT 

An object which is redundant with another object in the accessible hierarchy. ATs typically ignore objects with this role.

FORM 

A container of form controls. An example of the use of this role is to represent an html:form.

IME 

An object which is used to allow input of characters not found on a keyboard, such as the input of Chinese characters on a Western keyboard.

APP_ROOT 

XXX: document this.

PARENT_MENUITEM 

Represents a menu item, which is an entry in a menu that a user can choose to display another menu.

CALENDAR 

A calendar that allows the user to select a date.

COMBOBOX_LIST 

A list of items that is shown by combobox.

COMBOBOX_OPTION 

A item of list that is shown by combobox.

IMAGE_MAP 

An image map -- has child links representing the areas

OPTION 

An option in a listbox

RICH_OPTION 

A rich option in a listbox, it can have other widgets as children

LISTBOX 

A list of options

FLAT_EQUATION 

Represents a mathematical equation in the accessible name

GRID_CELL 

Represents a cell within a grid. It is used for role="gridcell". Unlike CELL, it allows the calculation of the accessible name from subtree. Also, see TABLE.

EMBEDDED_OBJECT 

Represents an embedded object. It is used for html:object or html:embed.

NOTE 

A note. Originally intended to be hidden until activated, but now also used for things like html 'aside'.

FIGURE 

A figure. Used for things like HTML5 figure element.

CHECK_RICH_OPTION 

Represents a rich item with a check box.

DEFINITION_LIST 

Represent a definition list (dl in HTML).

TERM 

Represent a term in a definition list (dt in HTML).

DEFINITION 

Represent a definition in a definition list (dd in HTML)

KEY 

Represent a keyboard or keypad key (ARIA role "key").
