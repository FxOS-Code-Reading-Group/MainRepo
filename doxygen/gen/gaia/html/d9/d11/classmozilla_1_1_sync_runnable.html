説明

This class will wrap a nsIRunnable and dispatch it to the main thread synchronously. This is different from nsIEventTarget.DISPATCH_SYNC: this class does not spin the event loop waiting for the event to be dispatched. This means that you don't risk reentrance from pending messages, but you must be sure that the target thread does not ever block on this thread, or else you will deadlock.

Typical usage: nsRefPtr<SyncRunnable> sr = new SyncRunnable(new myrunnable...()); sr->DispatchToThread(t);

We also provide a convenience wrapper: SyncRunnable::DispatchToThread(new myrunnable...());
