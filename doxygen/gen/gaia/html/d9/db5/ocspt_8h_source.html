00027 /*
00028  * This interface is described in terms of an HttpClient which
00029  * supports at least a specified set of functions. (An implementer may
00030  * provide HttpClients with additional functionality accessible only to
00031  * users with a particular implementation in mind.) The basic behavior
00032  * is provided by defining a set of functions, listed in an
00033  * SEC_HttpServerFcnStruct. If the implementor of a SpecificHttpClient
00034  * registers his SpecificHttpClient as the default HttpClient, then his
00035  * functions will be called by the user of an HttpClient, such as an
00036  * OCSPChecker.
00037  *
00038  * The implementer of a specific HttpClient (e.g., the NSS-provided
00039  * DefaultHttpClient), populates an SEC_HttpClientFcnStruct, uses it to
00040  * register his client, and waits for his functions to be called.
00041  *
00042  * For future expandability, the SEC_HttpClientFcnStruct is defined as a
00043  * union, with the version field acting as a selector. The proposed
00044  * initial version of the structure is given following the definition
00045  * of the union. The HttpClientState structure is implementation-
00046  * dependent, and should be opaque to the user.
00047  */

00153 /*
00154  * This function initiates or continues an HTTP request. After
00155  * parameters have been set with the Create function and, optionally,
00156  * modified or enhanced with the AddParams function, this call creates
00157  * the socket connection and initiates the communication.
00158  *
00159  * If a timeout value of zero is specified, indicating non-blocking
00160  * I/O, the client creates a non-blocking socket, and returns a status
00161  * of SECWouldBlock and a non-NULL PRPollDesc if the operation is not
00162  * complete. In that case all other return parameters are undefined.
00163  * The caller is expected to repeat the call, possibly after using
00164  * PRPoll to determine that a completion has occurred, until a return
00165  * value of SECSuccess (and a NULL value for pPollDesc) or a return
00166  * value of SECFailure (indicating failure on the network level)
00167  * is obtained.
00168  *
00169  * http_response_data_len is both input and output parameter.
00170  * If a pointer to a PRUint32 is supplied, the http client is
00171  * expected to check the given integer value and always set an out
00172  * value, even on failure.
00173  * An input value of zero means, the caller will accept any response len.
00174  * A different input value indicates the maximum response value acceptable
00175  * to the caller.
00176  * If data is successfully read and the size is acceptable to the caller,
00177  * the function will return SECSuccess and set http_response_data_len to
00178  * the size of the block returned in http_response_data.
00179  * If the data read from the http server is larger than the acceptable
00180  * size, the function will return SECFailure.
00181  * http_response_data_len will be set to a value different from zero to
00182  * indicate the reason of the failure.
00183  * An out value of "0" means, the failure was unrelated to the 
00184  * acceptable size.
00185  * An out value of "1" means, the result data is larger than the
00186  * accpeptable size, but the real size is not yet known to the http client 
00187  * implementation and it stopped retrieving it,
00188  * Any other out value combined with a return value of SECFailure
00189  * will indicate the actual size of the server data.
00190  *
00191  * The caller is permitted to provide NULL values for any of the
00192  * http_response arguments, indicating the caller is not interested in
00193  * those values. If the caller does provide an address, the HttpClient
00194  * stores at that address a pointer to the corresponding argument, at
00195  * the completion of the operation.
00196  *
00197  * All returned pointers will be owned by the the HttpClient
00198  * implementation and will remain valid until the call to 
00199  * SEC_HttpRequest_FreeFcn.
00200  */ 
00201 typedef SECStatus (*SEC_HttpRequest_TrySendAndReceiveFcn)(
00202    SEC_HTTP_REQUEST_SESSION request,
00203    PRPollDesc **pPollDesc,
00204    PRUint16 *http_response_code, 
00205    const char **http_response_content_type, 
00206    const char **http_response_headers, 
00207    const char **http_response_data, 
00208    PRUint32 *http_response_data_len);

00258 /*
00259  * ocspMode_FailureIsVerificationFailure:
00260  * This is the classic behaviour of NSS.
00261  * Any OCSP failure is a verification failure (classic mode, default).
00262  * Without a good response, OCSP networking will be retried each time
00263  * it is required for verifying a cert.
00264  *
00265  * ocspMode_FailureIsNotAVerificationFailure:
00266  * If we fail to obtain a valid OCSP response, consider the
00267  * cert as good.
00268  * Failed OCSP attempts might get cached and not retried until
