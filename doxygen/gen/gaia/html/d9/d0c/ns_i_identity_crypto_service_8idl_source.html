00011 /* Naming and calling conventions:
00012  *
00013  * A"hex" prefix means "hex-encoded string representation of a byte sequence"
00014  * e.g. "ae34bcdf123"
00015  *
00016  * A "base64url" prefix means "base-64-URL-encoded string repressentation of a
00017  * byte sequence.
00018  * e.g. "eyJhbGciOiJSUzI1NiJ9"
00019  * http://en.wikipedia.org/wiki/Base64#Variants_summary_table
00020  * we use the no-padding approach to base64-url-encoding
00021  *
00022  * Callbacks take an "in nsresult rv" argument that indicates whether the async
00023  * operation succeeded. On success, rv will be a success code
00024  * (NS_SUCCEEDED(rv) / Components.isSuccessCode(rv)) and the remaining
00025  * arguments are as defined in the documentation for the callback. When the
00026  * operation fails, rv will be a failure code (NS_FAILED(rv) /
00027  * !Components.isSuccessCode(rv)) and the values of the remaining arguments will
00028  * be unspecified.
00029  *
00030  * Key Types:
00031  *
00032  * "RS256": RSA + SHA-256.
00033  *
00034  * "DS160": DSA with SHA-1. A 1024-bit prime and a 160-bit subprime with SHA-1.
00035  *
00036  * we use these abbreviated algorithm names as per the JWA spec
00037  * http://tools.ietf.org/html/draft-ietf-jose-json-web-algorithms-02
00038  */
