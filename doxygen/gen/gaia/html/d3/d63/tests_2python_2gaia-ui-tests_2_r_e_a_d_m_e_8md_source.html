00001 Introduction
00002 ============
00003 
00004 Gaiatest is a Python package based on
00005 [Marionette](https://developer.mozilla.org/en-US/docs/Marionette), which is
00006 designed specifically for writing tests against
00007 [Gaia](https://github.com/mozilla-b2g/gaia).
00008 
00009 Prerequisites
00010 =============
00011 
00012 You will need a
00013 [Marionette enabled Firefox build](https://developer.mozilla.org/en-US/docs/Marionette/Builds)
00014 that you can
00015 [successfully connect to](https://developer.mozilla.org/en-US/docs/Marionette/Connecting_to_B2G).
00016 
00017 Installation
00018 ============
00019 
00020 Installation is simple:
00021 
00022     git clone git://github.com/mozilla/gaia-ui-tests.git
00023     cd gaia-ui-tests
00024     python setup.py develop
00025 
00026 Risks
00027 =====
00028 
00029 Please visit [this page](https://developer.mozilla.org/en-US/docs/Gaia_Test_Runner) to understand and acknowledge the risks involved when running these tests.
00030 
00031 Command line interface
00032 ======================
00033 
00034 A helpful command line tool is provided for interacting with Gaia. For full
00035 usage details run `gcli --help` and for help on a specific command use `gcli
00036 <command> --help`.
00037 
00038 For example, to unlock the device, set brightness to 100%, connect to an
00039 unsecured network, and launch the Settings app:
00040 
00041 ```bash
00042 $ gcli unlock
00043 $ gcli setsetting screen.brightness 1
00044 $ gcli connectwifi MozillaGuest
00045 $ gcli launchapp Settings
00046 ```
00047 
00048 Running Tests
00049 =============
00050 
00051 To run tests using gaia test, your command-line will vary a little bit
00052 depending on what device you're using.  The general format is:
00053 
00054     gaiatest [options] /path/to/test_foo.py
00055 
00056 Options:
00057 
00058     --emulator arm --homedir /path/to/emulator:  use these options to
00059         let Marionette launch an emulator for you in which to run a test
00060     --address <host>:<port>  use this option to run a test on an emulator
00061         which you've manually launched yourself, a real device, or a b2g
00062         desktop build.  If you've used port forwarding as described below,
00063         you'd specify --address localhost:2828
00064     --testvars= (see section below)
00065     --restart restart target instance between tests. This option will remove 
00066         the /data/local/indexedDB and /data/b2g/mozilla folders and restore the 
00067         device back to a common state
00068 
00069 Testing on a Device
00070 ===================
00071 
00072 You must run a build of B2G on the device that has Marionette enabled.
00073 The easiest way to do that is to grab a nightly `eng` build, like
00074 [this one for Unagi](https://pvtbuilds.mozilla.org/pub/mozilla.org/b2g/nightly/mozilla-b2g18-unagi-eng/latest/)
00075 (currently it requires a Mozilla LDAP login). Flash that to your device.
00076 
00077 You should not enable Remote Debugging manually on the device because
00078 there will be competing debuggers. See
00079 [bug 764913](https://bugzilla.mozilla.org/show_bug.cgi?id=764913).
00080 
00081 If you are running the tests on a device connected via ADB (Android Debug
00082 Bridge), you must additionally set up port forwarding from the device to your
00083 local machine. You can do this by running the command:
00084 
00085     adb forward tcp:2828 tcp:2828
00086 
00087 ADB is available in emulator packages under out/host/linux_x86/bin.
00088 Alternatively, it may be downloaded as part of the
00089 [Android SDK](http://developer.android.com/sdk/index.html).
00090 
00091 Testing on Desktop build
00092 ========================
00093 
00094 You can download the latest build of the desktop client from [this location](http://ftp.mozilla.org/pub/mozilla.org/b2g/nightly/latest-mozilla-b2g18/),
00095 but make sure you download the appropriate file for your operating system.
00096 
00097 Note : Unfortunately, due to [Bug 832469](https://bugzilla.mozilla.org/show_bug.cgi?id=832469) the nightly desktop builds do not currently work on Windows, so you will
00098 need either Mac or Linux to continue :
00099 
00100   * **Mac**: b2g-[VERSION].multi.mac64.dmg
00101   * **Linux (32bit)**: b2g-[VERSION].multi.linux-i686.tar.bz2
00102   * **Linux (64bit)**: b2g-[VERSION].multi.linux-x86_64.tar.bz2
00103 
00104 Note : If you do not have the operating systems installed on your machine, a virtual machine is fine as well.
00105 
00106 Once downloaded, you will need to extract the contents to a local folder. For the purposes of the rest
00107 of this guide, I’ll refer to this location as `$B2G_HOME`.
00108 
00109 
00110 Add the line `user_pref('marionette.force-local', true);` to your gaia/profile/user.js file, which on :
00111 
00112   * **Mac** is located in $B2G_HOME/B2G.app/Contents/MacOS
00113   * **Linux** is located in $B2G_HOME/b2g
00114 
00115 Because we’re running against the desktop client we must filter out all tests that are unsuitable. To run the tests, use the following command:
00116 
00117 `gaiatest --address=localhost:2828 --type=b2g-antenna-bluetooth-carrier-camera-sdcard-wifi-xfail gaiatest/tests/manifest.ini`
00118 
00119 You should then start to see the tests running.
00120 
00121 Test Types
00122 ==========
00123 Tests can be filtered by type, and the types are defined in the manifest files. Tests can belong to multiple types, some
00124 types imply others, and some are mutually exclusive - for example a test cannot be both 'online' and 'offline' but a
00125 test that is 'lan' is by definition 'online'. Be warned that despite these rules, there is no error checking on types,
00126 so you must take care when assigning them. Default types are set in the [DEFAULT] section of a manifest file, and are
00127 inherited by manifest files referenced by an include.
00128 
00129 Here is a list of the types used, and when to use them:
00130 
00131 * b2g - this means the test is a B2G (Firefox OS) test. All tests must include this type.
00132 * antenna - these tests require an antenna (headphones) to be connected.
00133 * bluetooth - requires bluetooth to be available.
00134 * camera - these tests require use of a camera.
00135 * carrier - an active SIM card with carrier connection is required.
00136 * lan - a local area connection (not cell data) is required by these tests (see note below).
00137 * offline - specifically requires no online connection.
00138 * online - some sort of online connection (lan or carrier) is required.
00139 * qemu - these tests require the Firefox OS emulator to run.
00140 * sdcard - a storage device must be present.
00141 * wifi - this means a WiFi connection is required.
00142 * xfail - a special type that indicates the test is expected to fail.
00143 
00144 You may be thinking that there is only WiFi or cell data, and why the need for the 'lan' test type. Well, these tests
00145 aren't only run on mobile devices... We also run then on single-board computers known as
00146 [pandaboards](https://en.wikipedia.org/wiki/Panda_Board), which have an ethernet port, and on desktop builds, which
00147 share the host computer's connection. It is for this reason that we need 'lan' to indicate a connection that is not cell
00148 data. For an example of where online/lan/carrier are used take a look at the browser tests.
00149 
00150 Test Variables
00151 ==============
00152 We use the --testvars option to pass in local variables, particularly those that cannot be checked into the repository. For example in gaia-ui-tests these variables can be your private login credentials, phone number or details of your WiFi connection.
00153 
00154 To use it, copy testvars_template.json to a different filename but add it into .gitignore so you don't check it into your repository.
00155 
00156 When running your tests add the argument:
00157     --testvars=(filename).json
00158 
00159 Variables:
00160 
00161 `"carrier": {} (dict)` The carrier information of the test phone. This contains the phone number, country and network of the SIM card.
00162 
00163 ```
00164 "carrier":{
00165     "phone_number": "",
00166     "country": "",
00167     "network": ""
00168 }
00169 ```
00170 `"imei": "" (string)` The 12 digit IMEI code of the test phone.
00171 `"remote_phone_number": "" (string)` A phone number that your device can call during the tests (try not to be a nuisance!). Prefix the number with '+' and your international dialing code.
00172 `"wifi":{}{ (dict)` This are the settings of your WiFi connection. Currently this supports WPA/WEP/etc. You can add WiFi networks by doing the following (remember to replace "KeyManagement" and "wep" with the value your network supports) :
00173 
00174 ```
00175 "wifi": {
00176     "ssid": "MyNetwork",
00177     "keyManagement": "WEP",
00178     "wep": "MyPassword"
00179 }
00180 ```
00181 
00182 WPA-PSK:
00183 ```
00184 "wifi": {
00185     "ssid": "MyNetwork",
00186     "keyManagement": "WPA-PSK",
00187     "psk": "MyPassword"
00188 }
00189 ```
00190 __Note__: Due to [Bug 775499](http://bugzil.la/775499), WiFi connections via WPA-EAP are not capable at this time.
00191 
00192 `"email": {} (dict)` The email login information used by the email tests. It can contain different types of email accounts:
00193 
00194 Gmail:
00195 ```
00196 "gmail": {
00197        "name": "",
00198        "email": "",
00199        "password": ""
00200     }
00201 ```
00202 
00203 Or different email protocols:
00204 ```
00205 "IMAP": {
00206     "name": "",
00207     "email": "",
00208     "password": "",
00209     "imap_hostname": "",
00210     "imap_name": "",
00211     "imap_port": "",
00212     "smtp_hostname": "",
00213     "smtp_name": "",
00214     "smtp_port": ""
00215 }
00216 ```
00217 Or:
00218 ```
00219 "ActiveSync":{
00220     "name": "",
00221     "email": "",
00222     "password": "",
00223     "active_sync_hostname": "",
00224     "active_sync_username": ""
00225 }
00226 ```
00227 `"settings": {} (dict)` Custom settings to override the Gaia default settings. These will be set before each test run but are not mandatory.
00228 ```
00229 "settings:{
00230     "<setting>":<value>
00231 }"
00232 ```

00244 Writing Tests
00245 =============
00246 
00247 Test writing for Marionette Python tests is described at
00248 https://developer.mozilla.org/en-US/docs/Marionette/Marionette_Python_Tests.
00249 
00250 Additionally, gaiatest exposes some API's for managing Gaia's lockscreen
00251 and application manager.  See https://github.com/mozilla-b2g/gaia/blob/master/tests/python/gaiatest/gaia_test.py.
00252 
00253 At the moment we don't have a specific style guide. Please follow the
00254 prevailing style of the existing tests. Use them as a template for writing
00255 your tests.
00256 We follow [PEP8](http://www.python.org/dev/peps/pep-0008/) for formatting, although we're pretty lenient on the
00257 80-character line length.
