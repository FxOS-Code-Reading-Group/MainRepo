00008 /* nsINativeAppSupport
00009  *
00010  * This "pseudo" (in the XPCOM sense) interface provides for
00011  * platform-specific general application support:
00012  *  o It manages the details of the simple DDE communication 
00013  *    supported on the Win32 platform (it is the addition of this 
00014  *    item that prompted the creation of this interface.
00015  *
00016  * Due to the nature of the beast, this interface is not a full-blown
00017  * XPCOM component.  The primary reason is that objects that implement
00018  * this interface generally must be operational *before* XPCOM (or any
00019  * of the rest of Mozilla) are initialized.  As a result, this 
00020  * interface is instantiated by somewhat unconventional means.
00021  *
00022  * To create the implementor of this interface, you call the function
00023  * NS_CreateNativeAppSupport.  This is done in the startup code
00024  * in nsAppRunner.cpp
00025  *
00026  * The interface provides these functions:
00027  *  start - You call this to inform the native app support that the  
00028  *          application is starting.  In addition, it serves as a
00029  *          query as to whether the application should continue to
00030  *          run.
00031  *
00032  *          If the returned boolean result is PR_FALSE, then the
00033  *          application should exit without further processing.  In
00034  *          such cases, the returned nsresult indicates whether the
00035  *          reason to exit is due to an error or not.
00036  *
00037  *          Win32 Note: In the case of starting a second instance
00038  *                      of this executable, this function will return
00039  *                      PR_FALSE and nsresult==NS_OK.  This means that
00040  *                      the command line arguments have been
00041  *                      successfully passed to the instance of the
00042  *                      application acting as a DDE server.
00043  *
00044  *  stop - You call this to inform the native app support that the
00045  *         application *wishes* to terminate.  If the returned boolean
00046  *         value is PR_FALSE, then the application should continue
00047  *         (as if there were still additional top-level windows open).
00048  *         
00049  *         Win32 Note: If this is the instance of the application
00050  *                     acting as the DDE server, and there are current
00051  *                     DDE conversations active with other instances
00052  *                     acting as DDE clients, then this function will
00053  *                     return PR_FALSE.
00054  * 
00055  *  quit - Like Stop, but this method *forces* termination (or more 
00056  *         precisely, indicates that the application is about to be
00057  *         terminated regardless of what a call to Stop might have
00058  *         returned.
00059  *
00060  *         This method is intended to be called when the user selects
00061  *         the "Quit" option (close all windows and exit).
00062  *
00063  *         Win32 Note: Stop is problematic in the case of "Quit" (close
00064  *                     all windows and exit the application) because
00065  *                     either we don't Quit or (potentially) we lose
00066  *                     requests coming from other instances of the
00067  *                     application.  The strategy is to give preference
00068  *                     to the user's explicit Quit request.  In the
00069  *                     unlikely event that a request is pending from
00070  *                     another instance of the application, then such
00071  *                     requests are essentially ignored.  This is
00072  *                     roughly equivalent to handling that request by
00073  *                     opening a new window, followed by immediately
00074  *                     closing it.  Since this is the same as if the
00075  *                     request came in immediately before the Quit
00076  *                     call (versus immediately after it), no harm.
00077  *
00078  *                     There is an exposure here: Upon return from this
00079  *                     function, any DDE connect request (for Mozilla)
00080  *                     will fail and other instances of the application
00081  *                     will start up as a DDE server.  In that case,
00082  *                     those instances may do things that conflict with
00083  *                     the subsequent shutting down of the instance that
00084  *                     is quitting.  For this reason, the call to Quit
00085  *                     should be deferred as long as possible.
00086  *
00087  *  onLastWindowClosing -  Called when the last window is closed. Used as a
00088  *                         "soft" shutdown, passwords are flushed.
00089  */
