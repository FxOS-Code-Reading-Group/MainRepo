00029 // A light-weight compression algorithm.  It is designed for speed of
00030 // compression and decompression, rather than for the utmost in space
00031 // savings.
00032 //
00033 // For getting better compression ratios when you are compressing data
00034 // with long repeated sequences or compressing data that is similar to
00035 // other data, while still compressing fast, you might look at first
00036 // using BMDiff and then compressing the output of BMDiff with
00037 // Snappy.
