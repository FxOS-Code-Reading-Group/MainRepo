00002 /*
00003  * The application is in charge of display detailed information about the usage.
00004  * Application has three tabs configured independently plus a settings view.
00005  *
00006  * This module is only in charge of manage the different tabs but the specific
00007  * behaviour is delegated to each tab (in /js/views directory).
00008  *
00009  * It is important to explain how the URL schema works. Cost Control
00010  * uses the concept of card to refer a tab or a view. There are two main layers:
00011  * the tab layer and the overlay layer. The first one is used to display tabs,
00012  * the second one is used to display settings or other overlaying dialogs.
00013  *
00014  * The URL schema is:
00015  * #tab-id[#overlay-id]
00016  *
00017  * So changing the hash to:
00018  * #balance-tab#settings-view
00019  * makes Cost Control to change to the balance view and show the settings.
00020  *
00021  * We can close the overlay layer by setting only the balance layer:
00022  * #datausage-tab
00023  * this makes Cost Control to change to the datausage tab at the same time it
00024  * closes the card in the overlay layer.
00025  *
00026  * If you want to preserve a layer but changing the other, use the empty string
00027  * as the id of the layer you want to preserve.
00028  * For instance, you want to only close the overlay layer but not affecting the
00029  * tab layer:
00030  * #
00031  *
00032  * Or you want to change tab layer but without affecting the overlay, use:
00033  * #telephony-tab#
00034  *
00035  * Remember: you cannot close the tab layer!
00036  */
