00016 /* 
00017 ** Win16 stdio special case.
00018 ** To get stdio to work for Win16, all calls to printf() and related
00019 ** things must be called from the environment of the .EXE; calls to
00020 ** printf() from the .DLL send output to the bit-bucket.
00021 **
00022 ** To make sure that PR_fprintf(), and related functions, work correctly,
00023 ** the actual stream I/O to stdout, stderr, stdin must be done in the
00024 ** .EXE. To do this, a hack is placed in _MD_Write() such that the
00025 ** fd for stdio handles results in a call to the .EXE.
00026 **
00027 ** file w16stdio.c contains the functions that get called from NSPR
00028 ** to do the actual I/O. w16stdio.o must be statically linked with
00029 ** any application needing stdio for Win16.
00030 **
00031 ** The address of these functions must be made available to the .DLL
00032 ** so he can call back to the .EXE. To do this, function 
00033 ** PR_MD_RegisterW16StdioCallbacks() is called from the .EXE.
00034 ** The arguments are the functions defined in w16stdio.c
00035 ** At runtime, MD_Write() calls the registered functions, if any
00036 ** were registered.
00037 **
00038 ** prinit.h contains a macro PR_STDIO_INIT() that calls the registration
00039 ** function for Win16; For other platforms, the macro is a No-Op.
00040 **
00041 ** Note that stdio is not operational at all on Win16 GUI applications.
00042 ** This special case exists to provide stdio capability from the NSPR
00043 ** .DLL for command line applications only. NSPR's test cases are
00044 ** almost exclusively command line applications.
00045 **
00046 ** See also: w16io.c, w16stdio.c
00047 */
00048 typedef PRInt32 (PR_CALLBACK *PRStdinRead)( void *buf, PRInt32 amount);
00049 typedef PRInt32 (PR_CALLBACK *PRStdoutWrite)( void *buf, PRInt32 amount);
00050 typedef PRInt32 (PR_CALLBACK *PRStderrWrite)( void *buf, PRInt32 amount);
