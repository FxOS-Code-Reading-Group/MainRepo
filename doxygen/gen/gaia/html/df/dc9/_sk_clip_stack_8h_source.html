00019 // stores the stack depth (fSaveCount) and clips (fDeque) separately.
00020 // Each clip in fDeque stores the stack state to which it belongs
00021 // (i.e., the fSaveCount in force when it was added). Restores are thus
00022 // implemented by removing clips from fDeque that have an fSaveCount larger

00168         /* fFiniteBoundType and fFiniteBound are used to incrementally update the clip stack's

00171            drawn to are inside the bound). When fFiniteBoundType is kInsideOut_BoundsType (which
00172            occurs when a clip is inverse filled), fFiniteBound represents the conservative bounding
00173            box of the pixels that _are_ clipped (i.e., any pixels that cannot be drawn to are inside
00174            the bound). When fFiniteBoundType is kInsideOut_BoundsType the actual bound is the
00175            infinite plane. This behavior of fFiniteBoundType and fFiniteBound is required so that we
