説明

This interface can be used to add a download to history. There is a separate interface specifically for downloads in case embedders choose to track downloads differently from other types of history.

Adds a download to history. This will also notify observers that the URI aSource is visited with the topic NS_LINK_VISITED_EVENT_TOPIC if aSource has not yet been visited.

引数:
aSourceThe source of the download we are adding to history. This cannot be null.
aReferrer[optional] The referrer of source URI.
aStartTime[optional] The time the download was started. If the start time is not given, the current time is used.
aDestination[optional] The target where the download is to be saved on the local filesystem.
例外:
NS_ERROR_NOT_AVAILABLEIn a situation where a history implementation is not available, where 'history implementation' refers to something like nsIGlobalHistory and friends.
覚え書き:
This addition is not guaranteed to be synchronous, since it delegates the actual addition to the underlying history implementation. If you need to observe the completion of the addition, use the underlying history implementation's notifications system (e.g. nsINavHistoryObserver for toolkit's implementation of this interface).
