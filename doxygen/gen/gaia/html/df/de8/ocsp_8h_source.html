00183 /*
00184  * FUNCTION: CERT_CreateOCSPRequest
00185  *   Creates a CERTOCSPRequest, requesting the status of the certs in 
00186  *   the given list.
00187  * INPUTS:
00188  *   CERTCertList *certList
00189  *     A list of certs for which status will be requested.
00190  *     Note that all of these certificates should have the same issuer,
00191  *     or it's expected the response will be signed by a trusted responder.
00192  *     If the certs need to be broken up into multiple requests, that
00193  *     must be handled by the caller (and thus by having multiple calls
00194  *     to this routine), who knows about where the request(s) are being
00195  *     sent and whether there are any trusted responders in place.
00196  *   PRTime time

00205  *     do not sign.
00206  *     XXX note that request signing is not yet supported; see comment in code
00207  * RETURN:
00208  *   A pointer to a CERTOCSPRequest structure containing an OCSP request
00209  *   for the cert list.  On error, null is returned, with an error set
00210  *   indicating the reason.  This is likely SEC_ERROR_UNKNOWN_ISSUER.
00211  *   (The issuer is needed to create a request for the certificate.)
00212  *   Other errors are low-level problems (no memory, bad database, etc.).
00213  */
00214 extern CERTOCSPRequest *
00215 CERT_CreateOCSPRequest(CERTCertList *certList, PRTime time, 
00216                PRBool addServiceLocator,
00217                CERTCertificate *signerCert);

00521  * FUNCTION: CERT_CacheOCSPResponseFromSideChannel
00522  *   First, this function checks the OCSP cache to see if a good response
00523  *   for the given certificate already exists. If it does, then the function
00524  *   returns successfully.
00525  *
00526  *   If not, then it validates that the given OCSP response is a valid,
00527  *   good response for the given certificate and inserts it into the
00528  *   cache.
00529  *
00530  *   This function is intended for use when OCSP responses are provided via a
00531  *   side-channel, i.e. TLS OCSP stapling (a.k.a. the status_request extension).
00532  *
00533  * INPUTS:
00534  *   CERTCertDBHandle *handle
00535  *     certificate DB of the cert that is being checked
00536  *   CERTCertificate *cert
00537  *     the certificate being checked
00538  *   PRTime time

00693  *   On error, returns null with error set describing the reason:
