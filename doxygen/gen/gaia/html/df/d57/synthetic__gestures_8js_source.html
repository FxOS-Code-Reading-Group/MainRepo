00002 // SyntheticGestures.js: test utilities for generating gesture events.
00003 //
00004 // This module defines a SyntheticGestures object whose properties are
00005 // utility functions that generate sequences of synthetic events. The event
00006 // sequences are intended to be recognized as gestures by GestureDetector.js
00007 // so this module is useful for writing tests of UI code that uses
00008 // GestureDetector.js.
00009 //
00010 // SyntheticGestures.tap(): emit touch events to generate a tap gesture
00011 //   on the specified target element.  GestureDetector.js should emit
00012 //   a tap event in response.  SyntheticGesture.mousetap() is the same
00013 //   but synthesizes mouse events instead.
00014 //
00015 // SyntheticGestures.dbltap(): emit touch events to generate a double
00016 //   tap gesture on the specified target element.  GestureDetector.js
00017 //   should emit two tap events and a dbltap event in response.
00018 //   SyntheticGesture.mousedbltap() is the same but synthesizes mouse
00019 //   events instead of touch events.
00020 //
00021 // SyntheticGestures.swipe(): emit touch events for a basic one finger
00022 //   touch, move, lift geture over a specifed target, with given start
00023 //   and end coordinates and gesture duration. GestureDetector.js should
00024 //   emit a series of pan events and a final swipe event.
00025 //   SyntheticGestures.mouseswipe() is similar but uses mouse events
00026 //   instead of touch events.
00027 //
00028 // SyntheticGestures.hold(): emit touch events for a touch/hold/move/release
00029 //   gesture. GestureDetector should emit a holdstart/holdmove/holdend
00030 //   sequence of events in response.  SyntheticGesture.mousehold() is
00031 //   similar but uses mouse events instead.
00032 //
00033 // SyntheticGestures.pinch(): emit touch events for a 2-finger pinch
00034 //   open or pinch close gesture with given staring points for each
00035 //   finger and a given scale factor. GestureDetector.js should
00036 //   respond with a sequence of transform events culminating in one
00037 //   with something very close to the specified scale factor.
00038 //   Two-finger gestures can't be simulated with a mouse, so there is no
00039 //   mouse-only version of this gesture, and the pinch() function will
00040 //   fail on platforms that do not support touch events.
00041 //
00042 // The coordinates passed to these functions are relative to the
00043 // coordinate system of the target element.  If you pass numbers, they
00044 // are interpreted as pixels relative to upper-left corner of the
00045 // target.  If you pass a string, then it is parsed to a number. If a
00046 // string-valued coordinate ends with '%', then it is interpreted as a
00047 // percent of the target element's width or height, otherwise it is
00048 // interpreted as a number of pixels. For the swipe(), mouseswipe(),
00049 // hold(), and mousehold() functions, if the second two coordinates
00050 // are strings that begin with "+" or "-" then they are interpreted
00051 // relative to the first two coordinates. These relative values can be
00052 // specifed as percentages or as pixels.
00053 //
