virtual nsresult nsExternalHelperAppService::GetFileTokenForPath ( const PRUnichar *  platformAppPath,
nsIFile **  aFile 
) [virtual]

Given a string identifying an application, create an nsIFile representing it. This function should look in $PATH for the application. The base class implementation will first try to interpret platformAppPath as an absolute path, and if that fails it will look for a file next to the mozilla executable. Subclasses can override this method if they want a different behaviour.

引数:
platformAppPathA platform specific path to an application that we got out of the rdf data source. This can be a mac file spec, a unix path or a windows path depending on the platform
aFile[out] An nsIFile representation of that platform application path.

virtual already_AddRefed<nsIMIMEInfo> nsExternalHelperAppService::GetMIMEInfoFromOS ( const nsACString &  aMIMEType,
const nsACString &  aFileExt,
bool *  aFound 
) [pure virtual]

Given a mimetype and an extension, looks up a mime info from the OS. The mime type is given preference. This function follows the same rules as nsIMIMEService::GetFromTypeAndExtension. This is supposed to be overridden by the platform-specific nsOSHelperAppService!

引数:
aFileExtThe file extension; may be empty. UTF-8 encoded.
[out]aFoundShould be set to true if the os has a mapping, to false otherwise. Must not be null.
戻り値:
A MIMEInfo. This function must return a MIMEInfo object if it can allocate one. The only justifiable reason for not returning one is an out-of-memory error. If null, the value of aFound is unspecified.
