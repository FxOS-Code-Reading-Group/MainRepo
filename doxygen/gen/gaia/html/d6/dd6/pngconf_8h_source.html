00098 /* Function calling conventions.
00099  * =============================
00100  * Normally it is not necessary to specify to the compiler how to call
00101  * a function - it just does it - however on x86 systems derived from
00102  * Microsoft and Borland C compilers ('IBM PC', 'DOS', 'Windows' systems
00103  * and some others) there are multiple ways to call a function and the
00104  * default can be changed on the compiler command line.  For this reason
00105  * libpng specifies the calling convention of every exported function and
00106  * every function called via a user supplied function pointer.  This is
00107  * done in this file by defining the following macros:
00108  *
00109  * PNGAPI    Calling convention for exported functions.
00110  * PNGCBAPI  Calling convention for user provided (callback) functions.
00111  * PNGCAPI   Calling convention used by the ANSI-C library (required
00112  *           for longjmp callbacks and sometimes used internally to
00113  *           specify the calling convention for zlib).
00114  *
00115  * These macros should never be overridden.  If it is necessary to
00116  * change calling convention in a private build this can be done
00117  * by setting PNG_API_RULE (which defaults to 0) to one of the values
00118  * below to select the correct 'API' variants.
00119  *
00120  * PNG_API_RULE=0 Use PNGCAPI - the 'C' calling convention - throughout.
00121  *                This is correct in every known environment.
00122  * PNG_API_RULE=1 Use the operating system convention for PNGAPI and
00123  *                the 'C' calling convention (from PNGCAPI) for
00124  *                callbacks (PNGCBAPI).  This is no longer required
00125  *                in any known environment - if it has to be used
00126  *                please post an explanation of the problem to the
00127  *                libpng mailing list.
00128  *
00129  * These cases only differ if the operating system does not use the C
00130  * calling convention, at present this just means the above cases
00131  * (x86 DOS/Windows sytems) and, even then, this does not apply to
00132  * Cygwin running on those systems.
00133  *
00134  * Note that the value must be defined in pnglibconf.h so that what
00135  * the application uses to call the library matches the conventions
00136  * set when building the library.
00137  */
00138 
00139 /* Symbol export
00140  * =============
00141  * When building a shared library it is almost always necessary to tell
00142  * the compiler which symbols to export.  The png.h macro 'PNG_EXPORT'
00143  * is used to mark the symbols.  On some systems these symbols can be
00144  * extracted at link time and need no special processing by the compiler,
00145  * on other systems the symbols are flagged by the compiler and just
00146  * the declaration requires a special tag applied (unfortunately) in a
00147  * compiler dependent way.  Some systems can do either.
00148  *
00149  * A small number of older systems also require a symbol from a DLL to
00150  * be flagged to the program that calls it.  This is a problem because
00151  * we do not know in the header file included by application code that
00152  * the symbol will come from a shared library, as opposed to a statically
00153  * linked one.  For this reason the application must tell us by setting
00154  * the magic flag PNG_USE_DLL to turn on the special processing before
00155  * it includes png.h.
00156  *
00157  * Four additional macros are used to make this happen:
00158  *
00159  * PNG_IMPEXP The magic (if any) to cause a symbol to be exported from
00160  *            the build or imported if PNG_USE_DLL is set - compiler
00161  *            and system specific.
00162  *
00163  * PNG_EXPORT_TYPE(type) A macro that pre or appends PNG_IMPEXP to
00164  *                       'type', compiler specific.
00165  *
00166  * PNG_DLL_EXPORT Set to the magic to use during a libpng build to
00167  *                make a symbol exported from the DLL.  Not used in the
00168  *                public header files; see pngpriv.h for how it is used
00169  *                in the libpng build.
00170  *
00171  * PNG_DLL_IMPORT Set to the magic to force the libpng symbols to come
00172  *                from a DLL - used to define PNG_IMPEXP when
00173  *                PNG_USE_DLL is set.
00174  */
00175
