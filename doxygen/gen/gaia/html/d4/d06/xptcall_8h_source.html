00064         // In theory, ptr could point anywhere. But in practice it always points

00066         // avoid the extra allocation we would incur if we were to use val.p.

00068         // Various parts of XPConnect assume that ptr==&val, so we enforce it

00071         // Since ptr always points to &val, the semantics of this flag are kind of
00072         // dumb, since the ptr field is unnecessary. But changing them would

00074         // the way they are.
