00203 // a non-XPCOM, refcounting array of XPCOM objects
00204 // used as a member variable or stack variable - this object is NOT
00205 // refcounted, but the objects that it holds are
00206 //
00207 // most of the read-only accessors like ObjectAt()/etc do NOT refcount
00208 // on the way out. This means that you can do one of two things:
00209 //
00210 // * does an addref, but holds onto a reference
00211 // nsCOMPtr<T> foo = array[i];
00212 //
00213 // * avoids the refcount, but foo might go stale if array[i] is ever
00214 // * modified/removed. Be careful not to NS_RELEASE(foo)!
00215 // T* foo = array[i];
00216 //
00217 // This array will accept null as an argument for any object, and will
00218 // store null in the array, just like nsVoidArray. But that also means
00219 // that methods like ObjectAt() may return null when referring to an
00220 // existing, but null entry in the array.
