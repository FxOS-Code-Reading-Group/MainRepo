00004 ## Running in Firefox
00005 
00006 Would you like to develop the calendar app in Firefox instead of b2g  
00007 desktop? Yes, you would.

00009 ### Step 1: Setup Gaia inside of Firefox
00010 
00011 If you have not already done so, follow the steps below to setup Gaia in Firefox.
00012 
00013 https://developer.mozilla.org/en-US/docs/Mozilla/Firefox_OS/Using_Gaia_in_Firefox
00014 
00015 ### Step 2: Run Gaia Calendar!
00016 
00017 Start the b2g-desktop instance to serve the calendar app on port 8080  
00018 with  
00019 
00020     cd /path/to/gaia
00021     DEBUG=1 make
00022     /path/to/nightly/firefox -jsconsole -profile profile/
00023 
00024 Browse to http://calendar.gaiamobile.org:8080/.

00031 ### Distribution and Customization
00032 
00033 A calendar.json file may be specified inside of distribution/calendar.json. This will override the generated presets.js file during the build. At a minimum Google Oauth credentials should be provided for production releases. Failure to do so may result in broken calendars at some point.
00034 
00035 You can signup for Google credentials here: https://code.google.com/apis/console/b/0/?pli=1#access
