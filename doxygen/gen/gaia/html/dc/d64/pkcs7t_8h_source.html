00027 /* Non-opaque objects.  NOTE, though: I want them to be treated as
00028  * opaque as much as possible.  If I could hide them completely,
00029  * I would.  (I tried, but ran into trouble that was taking me too
00030  * much time to get out of.)  I still intend to try to do so.
00031  * In fact, the only type that "outsiders" should even *name* is
00032  * SEC_PKCS7ContentInfo, and they should not reference its fields.
00033  */
00034 /* rjr: PKCS #11 cert handling (pk11cert.c) does use SEC_PKCS7RecipientInfo's.
00035  * This is because when we search the recipient list for the cert and key we
00036  * want, we need to invert the order of the loops we used to have. The old
00037  * loops were:
00038  *
00039  *  For each recipient {
00040  *       find_cert = PK11_Find_AllCert(recipient->issuerSN);
00041  *       [which unrolls to... ]
00042  *       For each slot {
00043  *            Log into slot;
00044  *            search slot for cert;
00045  *      }
00046  *  }
00047  *
00048  *  the new loop searchs all the recipients at once on a slot. this allows
00049  *  PKCS #11 to order slots in such a way that logout slots don't get checked
00050  *  if we can find the cert on a logged in slot. This eliminates lots of
00051  *  spurious password prompts when smart cards are installed... so why this
00052  *  comment? If you make SEC_PKCS7RecipientInfo completely opaque, you need
00053  *  to provide a non-opaque list of issuerSN's (the only field PKCS#11 needs
00054  *  and fix up pk11cert.c first. NOTE: Only S/MIME calls this special PKCS #11
00055  *  function.
00056  */
