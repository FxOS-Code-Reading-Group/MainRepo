00022 /*
00023  * RangedPtr is a smart pointer restricted to an address range specified at
00024  * creation.  The pointer (and any smart pointers derived from it) must remain
00025  * within the range [start, end] (inclusive of end to facilitate use as
00026  * sentinels).  Dereferencing or indexing into the pointer (or pointers derived
00027  * from it) must remain within the range [start, end).  All the standard pointer
00028  * operators are defined on it; in debug builds these operations assert that the
00029  * range specified at construction is respected.
00030  *
00031  * In theory passing a smart pointer instance as an argument can be slightly
00032  * slower than passing a T* (due to ABI requirements for passing structs versus
00033  * passing pointers), if the method being called isn't inlined.  If you are in
00034  * extremely performance-critical code, you may want to be careful using this
00035  * smart pointer as an argument type.
00036  *
00037  * RangedPtr<T> intentionally does not implicitly convert to T*.  Use get() to
00038  * explicitly convert to T*.  Keep in mind that the raw pointer of course won't
00039  * implement bounds checking in debug builds.
00040  */

00042 class RangedPtr
