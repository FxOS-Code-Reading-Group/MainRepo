00011 /******
00012 
00013   This file contains the list of all parsed CSS properties.  It is
00014   designed to be used as inline input through the magic of C
00015   preprocessing.  All entries must be enclosed in the appropriate
00016   CSS_PROP_* macro which will have cruel and unusual things done to it.
00017   It is recommended (but not strictly necessary) to keep all entries in
00018   alphabetical order.
00019 
00020   The arguments to CSS_PROP and CSS_PROP_* are:
00021 
00022   -. 'name' entries represent a CSS property name and *must* use only
00023   lowercase characters.
00024 
00025   -. 'id' should be the same as 'name' except that all hyphens ('-')
00026   in 'name' are converted to underscores ('_') in 'id'. For properties
00027   on a standards track, any '-moz-' prefix is removed in 'id'. This
00028   lets us do nice things with the macros without having to copy/convert
00029   strings at runtime.  These are the names used for the enum values of
00030   the nsCSSProperty enumeration defined in nsCSSProps.h.
00031 
00032   -. 'method' is designed to be as input for CSS2Properties and similar
00033   callers.  It must always be the same as 'name' except it must use
00034   InterCaps and all hyphens ('-') must be removed.  Callers using this
00035   parameter must also define the CSS_PROP_PUBLIC_OR_PRIVATE(publicname_,
00036   privatename_) macro to yield either publicname_ or privatename_.
00037   The names differ in that publicname_ has Moz prefixes where they are
00038   used, and also in CssFloat vs. Float.  The caller's choice depends on
00039   whether the use is for internal use such as eCSSProperty_* or
00040   nsRuleData::ValueFor* or external use such as exposing DOM properties.
00041 
00042   -. 'pref' is the name of a pref that controls whether the property
00043   is enabled.  The property is enabled if 'pref' is an empty string,
00044   or if the boolean property whose name is 'pref' is set to true.
00045 
00046   -. 'flags', a bitfield containing CSS_PROPERTY_* flags.
00047 
00048   -. 'parsevariant', to be passed to ParseVariant in the parser.
00049 
00050   -. 'kwtable', which is either nullptr or the name of the appropriate
00051   keyword table member of class nsCSSProps, for use in
00052   nsCSSProps::LookupPropertyValue.
00053 
00054   -. 'stylestruct_' [used only for CSS_PROP, not CSS_PROP_*] gives the
00055   name of the style struct.  Can be used to make nsStyle##stylestruct_
00056   and eStyleStruct_##stylestruct_
00057 
00058   -. 'stylestructoffset_' [not used for CSS_PROP_BACKENDONLY] gives the
00059   result of offsetof(nsStyle*, member).  Ignored (and generally
00060   CSS_PROP_NO_OFFSET, or -1) for properties whose animtype_ is
00061   eStyleAnimType_None.
00062 
00063   -. 'animtype_' [not used for CSS_PROP_BACKENDONLY] gives the
00064   animation type (see nsStyleAnimType) of this property.
00065 
00066   CSS_PROP_SHORTHAND only takes 1-5.
00067 
00068  ******/

00235 // For notes XXX bug 3935 below, the names being parsed do not correspond
00236 // to the constants used internally.  It would be nice to bring the
00237 // constants into line sometime.
00238 
00239 // The parser will refuse to parse properties marked with -x-.
00240 
00241 // Those marked XXX bug 48973 are CSS2 properties that we support
00242 // differently from the spec for UI requirements.  If we ever
00243 // support them correctly the old constants need to be renamed and
00244 // new ones should be entered.
00245 
00246 // CSS2.1 section 5.12.1 says that the properties that apply to
00247 // :first-line are: font properties, color properties, background
00248 // properties, 'word-spacing', 'letter-spacing', 'text-decoration',
00249 // 'vertical-align', 'text-transform', and 'line-height'.
00250 //
00251 // We also allow 'text-shadow', which was listed in CSS2 (where the
00252 // property existed).
00253 
00254 // CSS2.1 section 5.12.2 says that the properties that apply to
00255 // :first-letter are: font properties, 'text-decoration',
00256 // 'text-transform', 'letter-spacing', 'word-spacing' (when
00257 // appropriate), 'line-height', 'float', 'vertical-align' (only if
00258 // 'float' is 'none'), margin properties, padding properties, border
00259 // properties, 'color', and background properties.  We also allow
00260 // 'text-shadow' (see above) and 'box-shadow' (which is like the
00261 // border properties).
00262 
00263 // We include '-moz-background-inline-policy' (css3-background's
00264 // 'background-break') in both as a background property, although this
00265 // is somewhat questionable.
