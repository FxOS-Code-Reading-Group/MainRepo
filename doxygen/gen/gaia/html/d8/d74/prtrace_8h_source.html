00008 /*
00009 ** prtrace.h -- NSPR's Trace Facility.                     
00010 **                                                                                 
00011 ** The Trace Facility provides a means to trace application                        
00012 ** program events within a process. When implementing an                           
00013 ** application program an engineer may insert a "Trace" function                   
00014 ** call, passing arguments to be traced. The "Trace" function                      
00015 ** combines the user trace data with identifying data and                          
00016 ** writes this data in time ordered sequence into a circular                       
00017 ** in-memory buffer; when the buffer fills, it wraps.
00018 **                                                                                 
00019 ** Functions are provided to set and/or re-configure the size of                   
00020 ** the trace buffer, control what events are recorded in the                       
00021 ** buffer, enable and disable tracing based on specific user                       
00022 ** supplied data and other control functions. Methods are provided                 
00023 ** to record the trace entries in the in-memory trace buffer to
00024 ** a file.
00025 **                                                                                 
00026 ** Tracing may cause a performance degredation to the application                  
00027 ** depending on the number and placement of calls to the tracing                   
00028 ** facility. When tracing is compiled in and all tracing is                        
00029 ** disabled via the runtime controls, the overhead should be                       
00030 ** minimal. ... Famous last words, eh?                                             
00031 **                                                                                 
00032 ** When DEBUG is defined at compile time, the Trace Facility is                    
00033 ** compiled as part of NSPR and any application using NSPR's                       
00034 ** header files will have tracing compiled in. When DEBUG is not                   
00035 ** defined, the Trace Facility is not compiled into NSPR nor                       
00036 ** exported in its header files.  If the Trace Facility is                         
00037 ** desired in a non-debug build, then FORCE_NSPR_TRACE may be                      
00038 ** defined at compile time for both the optimized build of NSPR                    
00039 ** and the application. NSPR and any application using  NSPR's                     
00040 ** Trace Facility must be compiled with the same level of trace                    
00041 ** conditioning or unresolved references may be realized at link                   
00042 ** time.                                                                           
00043 **                                                                                 
00044 ** For any of the Trace Facility methods that requires a trace                     
00045 ** handle as an input argument, the caller must ensure that the                    
00046 ** trace handle argument is valid. An invalid trace handle                         
00047 ** argument may cause unpredictable results.                                       
00048 **                                                                                 
00049 ** Trace Facility methods are thread-safe and SMP safe.                            
00050 **                                                                                 
00051 ** Users of the Trace Facility should use the defined macros to                     
00052 ** invoke trace methods, not the function calls directly. e.g.                      
00053 ** PR_TRACE( h1,0,1,2, ...); not PR_Trace(h1,0,1,2, ...);
00054 **                                                                                  
00055 ** Application designers should be aware of the effects of
00056 ** debug and optimized build differences when using result of the
00057 ** Trace Facility macros in expressions.
00058 ** 
00059 ** See Also: prcountr.h                                                                                 
00060 **                                                                                  
00061 ** /lth. 08-Jun-1998.                                                                                  
00062 */

00264 /* -----------------------------------------------------------------------
00265 ** FUNCTION: PR_SetTraceOption() -- Control the Trace Facility
00266 ** 
00267 ** DESCRIPTION:
00268 ** PR_SetTraceOption() controls the Trace Facility. Depending on
00269 ** command and value, attributes of the Trace Facility may be
00270 ** changed.
00271 ** 
00272 ** INPUTS:
00273 **  command: An enumerated value in the set of PRTraceOption.
00274 **  value: pointer to the data to be set. Type of the data is
00275 **  dependent on command; for each value of command, the type
00276 **  and meaning of dereferenced value is shown.
00277 **
00278 **  PRTraceBufSize: unsigned long: the size of the trace buffer,
00279 ** in bytes.
00280 ** 
00281 **  PRTraceEnable: PRTraceHandle. The trace handle to be
00282 ** enabled.
00283 ** 
00284 **  PRTraceDisable: PRTraceHandle. The trace handle to be
00285 ** disabled.
00286 ** 
00287 **  PRTraceSuspend: void. value must be NULL. All tracing is
00288 ** suspended.
00289 ** 
00290 **  PRTraceResume: void. value must be NULL. Tracing for all
00291 ** previously enabled, prior to a PRTraceSuspend, is resumed.
00292 ** 
00293 **  PRTraceStopRecording: void. value must be NULL. If recording
00294 ** (see: ** PR_RecordTraceEntries()) is being done, 
00295 ** PRTraceStopRecording causes PR_RecordTraceEntries() to return
00296 ** to its caller. If recording is not being done, this function
00297 ** has no effect.
00298 ** 
00299 **  PRTraceSuspendRecording: void. Must be NULL. If recording is
00300 ** being done, PRTraceSuspendRecording causes further writes to
00301 ** the trace file to be suspended. Data in the in-memory
00302 ** trace buffer that would ordinarily be written to the
00303 ** trace file will not be written. Trace entries will continue
00304 ** to be entered in the in-memory buffer. If the Trace Facility
00305 ** recording is already in a suspended state, the call has no
00306 ** effect.
00307 ** 
00308 **  PRTraceResumeRecording: void. value must be NULL. If
00309 ** recording for the Trace Facility has been previously been
00310 ** suspended, this causes recording to resume. Recording resumes
00311 ** with the next in-memory buffer segment that would be written
00312 ** if trace recording had not been suspended. If recording is
00313 ** not currently suspended, the call has no effect.
00314 ** 
00315 **  PRTraceLockHandles: void. value must be NULL. Locks the
00316 ** trace handle lock. While the trace handle lock is held,
00317 ** calls to PR_CreateTrace() will block until the lock is
00318 ** released.
00319 ** 
00320 **  PRTraceUnlockHandles: void. value must be NULL. Unlocks the
00321 ** trace handle lock.
00322 ** 
00323 ** OUTPUTS:
00324 **  The operation of the Trace Facility may be changed.
00325 ** 
00326 ** RETURNS: void
00327 ** 
00328 ** RESTRICTIONS:
00329 ** 
00330 */

00534 /* -----------------------------------------------------------------------
00535 ** FUNCTION: PR_RecordTraceEntries() -- Write trace entries to external media
00536 ** 
00537 ** DESCRIPTION:
00538 ** PR_RecordTraceEntries() causes entries in the in-memory trace
00539 ** buffer to be written to external media.
00540 **
00541 ** When PR_RecordTraceEntries() is called from an application
00542 ** thread, the function appears to block until another thread
00543 ** calls PR_SetTraceOption() with the PRTraceStopRecording
00544 ** option. This suggests that PR_RecordTraceEntries() should be
00545 ** called from a user supplied thread whose only job is to
00546 ** record trace entries. 
00547 ** 
00548 ** The environment variable NSPR_TRACE_LOG controls the operation
00549 ** of this function. When NSPR_TRACE_LOG is not defined in the
00550 ** environment, no recording of trace entries occurs. When
00551 ** NSPR_TRACE_LOG is defined, the value of its definition must be
00552 ** the filename of the file to receive the trace entry buffer.
00553 **
00554 ** PR_RecordTraceEntries() attempts to record the in-memory
00555 ** buffer to a file, subject to the setting of the environment
00556 ** variable NSPR_TRACE_LOG. It is possible because of system
00557 ** load, the thread priority of the recording thread, number of
00558 ** active trace records being written over time, and other
00559 ** variables that some trace records can be lost. ... In other
00560 ** words: don't bet the farm on getting everything.
00561 ** 
00562 ** INPUTS: none
00563 ** 
00564 ** OUTPUTS: none
00565 ** 
00566 ** RETURNS: PR_STATUS
00567 **    PR_SUCCESS no errors were found.
00568 **    PR_FAILURE errors were found.
00569 ** 
00570 ** RESTRICTIONS:
00571 ** Only one thread can call PR_RecordTraceEntries() within a
00572 ** process.
00573 ** 
00574 ** On error, PR_RecordTraceEntries() may return prematurely.
00575 ** 
00576 */

00589 /* -----------------------------------------------------------------------

00591 ** the Trace Facility
00592 ** 
00593 ** DESCRIPTION:
00594 ** PR_GetTraceEntries() retreives trace entries from the Trace
00595 ** Facility. Up to count trace entries are copied from the Trace
00596 ** Facility into buffer. Only those trace entries that have not
00597 ** been copied via a previous call to PR_GetTraceEntries() are
00598 ** copied. The actual number copied is placed in the PRInt32
00599 ** variable pointed to by found.
00600 **
00601 ** If more than count trace entries have entered the Trace
00602 ** Facility since the last call to PR_GetTraceEntries() 
00603 ** a lost data condition is returned. In this case, the most
00604 ** recent count trace entries are copied into buffer and found is
00605 ** set to count.
00606 ** 
00607 ** INPUTS:
00608 **  count. The number of trace entries to be copied into buffer.
00609 ** 
00610 ** 
00611 ** OUTPUTS:
00612 **  buffer. An array of PRTraceEntries. The buffer is supplied
00613 ** by the caller.
00614 ** 
00615 ** found: 32bit signed integer. The number of PRTraceEntries
00616 ** actually copied. found is always less than or equal to count.
00617 ** 
00618 ** RETURNS: 
00619 **  zero when there is no lost data.
00620 **  non-zero when some PRTraceEntries have been lost.
00621 ** 
00622 ** RESTRICTIONS:
00623 ** This is a real performance pig. The copy out operation is bad
00624 ** enough, but depending on then frequency of calls to the
00625 ** function, serious performance impact to the operating
00626 ** application may be realized. ... YMMV.
00627 ** 
00628 */
