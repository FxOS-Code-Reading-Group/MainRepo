00011 /*
00012  Some components need to be run at the startup of mozilla or embedding - to 
00013  start new services etc.
00014 
00015  This interface provides a generic way to start up arbitrary components 
00016  without requiring them to hack into main1() (or into NS_InitEmbedding) as 
00017  it's currently being done for services such as wallet, command line handlers 
00018  etc.
00019 
00020  We will have a category called "app-startup" which components register 
00021  themselves in using the CategoryManager.
00022 
00023  Components can also (optionally) add the word "service," as a prefix 
00024  to the "value" they pass in during a call to AddCategoryEntry() as
00025  shown below:
00026 
00027     categoryManager->AddCategoryEntry(APPSTARTUP_CATEGORY, "testcomp",
00028                         "service," NS_WALLETSERVICE_CONTRACTID
00029                         true, true,
00030                         getter_Copies(previous));
00031 
00032  Presence of the "service" keyword indicates the components desire to 
00033  be started as a service. When the "service" keyword is not present
00034  we just do a do_CreateInstance.
00035 
00036  When mozilla starts (and when NS_InitEmbedding()) is invoked
00037  we create an instance of the AppStartupNotifier component (which 
00038  implements nsIObserver) and invoke its Observe() method. 
00039 
00040  Observe()  will enumerate the components registered into the
00041  APPSTARTUP_CATEGORY and notify them that startup has begun
00042  and release them.
00043 */
