00153 //  nsSmallVoidArray is not a general-purpose replacement for
00154 //  ns(Auto)VoidArray because there is (some) extra CPU overhead for arrays
00155 //  larger than 1 element, though not a lot.  It is appropriate for
00156 //  space-sensitive uses where sizes of 0 or 1 are moderately common or
00157 //  more, and where we're NOT storing arbitrary integers or arbitrary
00158 //  pointers.
00159 
00160 // NOTE: nsSmallVoidArray can ONLY be used for holding items that always
00161 // have the low bit as a 0 - i.e. element & 1 == 0.  This happens to be
00162 // true for allocated and object pointers for all the architectures we run
00163 // on, but conceivably there might be some architectures/compilers for
00164 // which it is NOT true.  We know this works for all existing architectures
00165 // because if it didn't then nsCheapVoidArray would have failed.  Also note
