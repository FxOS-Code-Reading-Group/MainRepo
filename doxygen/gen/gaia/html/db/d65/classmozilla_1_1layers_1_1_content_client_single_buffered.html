説明

A single buffered ContentClient. We have a single TextureClient/Host which we update and then send a message to the compositor that we are done updating. It is not safe for the compositor to use the corresponding TextureHost's memory directly, it must upload it to video memory of some kind. We are free to modify the TextureClient once we receive reply from the compositor.
