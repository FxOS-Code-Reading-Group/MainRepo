00037 /* An OAuth message is represented as an object like this:
00038    {method: "GET", action: "http://server.com/path", parameters: ...}
00039 
00040    The parameters may be either a map {name: value, name2: value2}
00041    or an Array of name-value pairs [[name, value], [name2, value2]].
00042    The latter representation is more powerful: it supports parameters
00043    in a specific sequence, or several parameters with the same name;
00044    for example [["a", 1], ["b", 2], ["a", 3]].
00045 
00046    Parameter names and values are NOT percent-encoded in an object.
00047    They must be encoded before transmission and decoded after reception.
00048    For example, this message object:
00049    {method: "GET", action: "http://server/path", parameters: {p: "x y"}}
00050    ... can be transmitted as an HTTP request that begins:
00051    GET /path?p=x%20y HTTP/1.0
00052    (This isn't a valid OAuth request, since it lacks a signature etc.)
00053    Note that the object "x y" is transmitted as x%20y.  To encode
00054    parameters, you can call OAuth.addToURL, OAuth.formEncode or
00055    OAuth.getAuthorization.
00056 
00057    This message object model harmonizes with the browser object model for
00058    input elements of an form, whose value property isn't percent encoded.
00059    The browser encodes each value before transmitting it. For example,
00060    see consumer.setInputs in example/consumer.js.
00061  */
