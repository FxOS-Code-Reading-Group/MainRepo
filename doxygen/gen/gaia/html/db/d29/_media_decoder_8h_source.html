00006 /*
00007 Each video element based on MediaDecoder has a state machine to manage
00008 its play state and keep the current frame up to date. All state machines
00009 share time in a single shared thread. Each decoder also has one thread
00010 dedicated to decoding audio and video data. This thread is shutdown when
00011 playback is paused. Each decoder also has a thread to push decoded audio
00012 to the hardware. This thread is not created until playback starts, but
00013 currently is not destroyed when paused, only when playback ends.
00014 
00015 The decoder owns the resources for downloading the media file, and the
00016 high level state. It holds an owning reference to the state machine that
00017 owns all the resources related to decoding data, and manages the low level
00018 decoding operations and A/V sync.
00019 
00020 Each state machine runs on the shared state machine thread. Every time some
00021 action is required for a state machine, it is scheduled to run on the shared
00022 the state machine thread. The state machine runs one "cycle" on the state
00023 machine thread, and then returns. If necessary, it will schedule itself to
00024 run again in future. While running this cycle, it must not block the
00025 thread, as other state machines' events may need to run. State shared
00026 between a state machine's threads is synchronised via the monitor owned
00027 by its MediaDecoder object.
00028 
00029 The Main thread controls the decode state machine by setting the value
00030 of a mPlayState variable and notifying on the monitor based on the
00031 high level player actions required (Seek, Pause, Play, etc).
00032 
00033 The player states are the states requested by the client through the
00034 DOM API.  They represent the desired state of the player, while the
00035 decoder's state represents the actual state of the decoder.
00036 
00037 The high level state of the player is maintained via a PlayState value.
00038 It can have the following states:
00039 
00040 START
00041   The decoder has been initialized but has no resource loaded.
00042 PAUSED
00043   A request via the API has been received to pause playback.
00044 LOADING
00045   A request via the API has been received to load a resource.
00046 PLAYING
00047   A request via the API has been received to start playback.
00048 SEEKING
00049   A request via the API has been received to start seeking.
00050 COMPLETED
00051   Playback has completed.
00052 SHUTDOWN
00053   The decoder is about to be destroyed.
00054 
00055 State transition occurs when the Media Element calls the Play, Seek,
00056 etc methods on the MediaDecoder object. When the transition
00057 occurs MediaDecoder then calls the methods on the decoder state
00058 machine object to cause it to behave as required by the play state.
00059 State transitions will likely schedule the state machine to run to
00060 affect the change.
00061 
00062 An implementation of the MediaDecoderStateMachine class is the event
00063 that gets dispatched to the state machine thread. Each time the event is run,
00064 the state machine must cycle the state machine once, and then return.
00065 
00066 The state machine has the following states:
00067 
00068 DECODING_METADATA
00069   The media headers are being loaded, and things like framerate, etc are
00070   being determined, and the first frame of audio/video data is being decoded.
00071 DECODING
00072   The decode has started. If the PlayState is PLAYING, the decode thread
00073   should be alive and decoding video and audio frame, the audio thread
00074   should be playing audio, and the state machine should run periodically
00075   to update the video frames being displayed.
00076 SEEKING
00077   A seek operation is in progress. The decode thread should be seeking.
00078 BUFFERING
00079   Decoding is paused while data is buffered for smooth playback. If playback
00080   is paused (PlayState transitions to PAUSED) we'll destory the decode thread.
00081 COMPLETED
00082   The resource has completed decoding, but possibly not finished playback.
00083   The decode thread will be destroyed. Once playback finished, the audio
00084   thread will also be destroyed.
00085 SHUTDOWN
00086   The decoder object and its state machine are about to be destroyed.
00087   Once the last state machine has been destroyed, the shared state machine
00088   thread will also be destroyed. It will be recreated later if needed.
00089 
00090 The following result in state transitions.
00091 
00092 Shutdown()
00093   Clean up any resources the MediaDecoderStateMachine owns.
00094 Play()
00095   Start decoding and playback of media data.
00096 Buffer
00097   This is not user initiated. It occurs when the
00098   available data in the stream drops below a certain point.
00099 Complete
00100   This is not user initiated. It occurs when the
00101   stream is completely decoded.
00102 Seek(double)
00103   Seek to the time position given in the resource.
00104 
00105 A state transition diagram:
00106 
00107 DECODING_METADATA
00108   |      |
00109   v      | Shutdown()
00110   |      |
00111   v      -->-------------------->--------------------------|
00112   |---------------->----->------------------------|        v
00113 DECODING             |          |  |              |        |
00114   ^                  v Seek(t)  |  |              |        |
00115   |         Play()   |          v  |              |        |
00116   ^-----------<----SEEKING      |  v Complete     v        v
00117   |                  |          |  |              |        |
00118   |                  |          |  COMPLETED    SHUTDOWN-<-|
00119   ^                  ^          |  |Shutdown()    |
00120   |                  |          |  >-------->-----^
00121   |          Play()  |Seek(t)   |Buffer()         |
00122   -----------<--------<-------BUFFERING           |
00123                                 |                 ^
00124                                 v Shutdown()      |
00125                                 |                 |
00126                                 ------------>-----|
00127 
00128 The following represents the states that the MediaDecoder object
00129 can be in, and the valid states the MediaDecoderStateMachine can be in at that
00130 time:
00131 
00132 player LOADING   decoder DECODING_METADATA
00133 player PLAYING   decoder DECODING, BUFFERING, SEEKING, COMPLETED
00134 player PAUSED    decoder DECODING, BUFFERING, SEEKING, COMPLETED
00135 player SEEKING   decoder SEEKING
00136 player COMPLETED decoder SHUTDOWN
00137 player SHUTDOWN  decoder SHUTDOWN
00138 
00139 The general sequence of events is:
00140 
00141 1) The video element calls Load on MediaDecoder. This creates the
00142    state machine and starts the channel for downloading the
00143    file. It instantiates and schedules the MediaDecoderStateMachine. The
00144    high level LOADING state is entered, which results in the decode
00145    thread being created and starting to decode metadata. These are
00146    the headers that give the video size, framerate, etc. Load() returns
00147    immediately to the calling video element.
00148 
00149 2) When the metadata has been loaded by the decode thread, the state machine
00150    will call a method on the video element object to inform it that this
00151    step is done, so it can do the things required by the video specification
00152    at this stage. The decode thread then continues to decode the first frame
00153    of data.
00154 
00155 3) When the first frame of data has been successfully decoded the state
00156    machine calls a method on the video element object to inform it that
00157    this step has been done, once again so it can do the required things
00158    by the video specification at this stage.
00159 
00160    This results in the high level state changing to PLAYING or PAUSED
00161    depending on any user action that may have occurred.
00162 
00163    While the play state is PLAYING, the decode thread will decode
00164    data, and the audio thread will push audio data to the hardware to
00165    be played. The state machine will run periodically on the shared
00166    state machine thread to ensure video frames are played at the
00167    correct time; i.e. the state machine manages A/V sync.
00168 
00169 The Shutdown method on MediaDecoder closes the download channel, and
00170 signals to the state machine that it should shutdown. The state machine
00171 shuts down asynchronously, and will release the owning reference to the
00172 state machine once its threads are shutdown.
00173 
00174 The owning object of a MediaDecoder object *MUST* call Shutdown when
00175 destroying the MediaDecoder object.
00176 
00177 */
