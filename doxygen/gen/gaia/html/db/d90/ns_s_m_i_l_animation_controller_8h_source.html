00031 // nsSMILAnimationController
00032 //
00033 // The animation controller maintains the animation timer and determines the
00034 // sample times and sample rate for all SMIL animations in a document. There is
00035 // at most one animation controller per nsDocument so that frame-rate tuning can
00036 // be performed at a document-level.
00037 //
00038 // The animation controller can contain many child time containers (timed
00039 // document root objects) which may correspond to SVG document fragments within
00040 // a compound document. These time containers can be paused individually or
00041 // here, at the document level.
00042 //

00192   // Average time between samples from the refresh driver. This is used to
00193   // detect large unexpected gaps between samples such as can occur when the
00194   // computer sleeps. The nature of the SMIL model means that catching up these
00195   // large gaps can be expensive as, for example, many events may need to be
00196   // dispatched for the intervening time when no samples were received.
00197   //
00198   // In such cases, we ignore the intervening gap and continue sampling from
00199   // when we were expecting the next sample to arrive.
00200   //
00201   // Note that we only do this for SMIL and not CSS transitions (which doesn't
00202   // have so much work to do to catch up) nor scripted animations (which expect
00203   // animation time to follow real time).
00204   //
00205   // This behaviour does not affect pausing (since we're not *expecting* any
00206   // samples then) nor seeking (where the SMIL model behaves somewhat
00207   // differently such as not dispatching events).
