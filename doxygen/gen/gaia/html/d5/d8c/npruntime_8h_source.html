00042 /*
00043     This API is used to facilitate binding code written in C to script
00044     objects.  The API in this header does not assume the presence of a
00045     user agent.  That is, it can be used to bind C code to scripting
00046     environments outside of the context of a user agent.
00047 
00048     However, the normal use of the this API is in the context of a
00049     scripting environment running in a browser or other user agent.
00050     In particular it is used to support the extended Netscape
00051     script-ability API for plugins (NP-SAP).  NP-SAP is an extension
00052     of the Netscape plugin API.  As such we have adopted the use of
00053     the "NP" prefix for this API.
00054 
00055     The following NP{N|P}Variables were added to the Netscape plugin
00056     API (in npapi.h):
00057 
00058     NPNVWindowNPObject
00059     NPNVPluginElementNPObject
00060     NPPVpluginScriptableNPObject
00061 
00062     These variables are exposed through NPN_GetValue() and
00063     NPP_GetValue() (respectively) and are used to establish the
00064     initial binding between the user agent and native code.  The DOM
00065     objects in the user agent can be examined and manipulated using
00066     the NPN_ functions that operate on NPObjects described in this
00067     header.
00068 
00069     To the extent possible the assumptions about the scripting
00070     language used by the scripting environment have been minimized.
00071 */

00272 /*
00273     NPObjects returned by create, retain, invoke, and getProperty pass
00274     a reference count to the caller.  That is, the callee adds a
00275     reference count which passes to the caller.  It is the caller's
00276     responsibility to release the returned object.
00277 
00278     NPInvokeFunctionPtr function may return 0 to indicate a void
00279     result.
00280 
00281     NPInvalidateFunctionPtr is called by the scripting environment
00282     when the native code is shutdown.  Any attempt to message a
00283     NPObject instance after the invalidate callback has been
00284     called will result in undefined behavior, even if the native code
00285     is still retaining those NPObject instances.  (The runtime
00286     will typically return immediately, with 0 or NULL, from an attempt
00287     to dispatch to a NPObject, but this behavior should not be
00288     depended upon.)
00289 
00290     The NPEnumerationFunctionPtr function may pass an array of
00291     NPIdentifiers back to the caller. The callee allocs the memory of
00292     the array using NPN_MemAlloc(), and it's the caller's responsibility
00293     to release it using NPN_MemFree().
00294 */
