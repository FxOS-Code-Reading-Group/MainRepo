00184 /* FUNCTION: PR_NTFast_AcceptRead_WithTimeoutCallback
00185 ** DESCRIPTION:
00186 **    The AcceptEx call combines the accept with the read function.  However,
00187 **    our daemon threads need to be able to wakeup and reliably flush their
00188 **    log buffers if the Accept times out.  However, with the current blocking
00189 **    interface to AcceptRead, there is no way for us to timeout the Accept;
00190 **    this is because when we timeout the Read, we can close the newly 
00191 **    socket and continue; but when we timeout the accept itself, there is no
00192 **    new socket to timeout.  So instead, this version of the function is
00193 **    provided.  After the initial timeout period elapses on the accept()
00194 **    portion of the function, it will call the callback routine and then
00195 **    continue the accept.   If the timeout occurs on the read, it will 
00196 **    close the connection and return error.
00197 */
00198 NSPR_API(PRInt32) PR_NTFast_AcceptRead_WithTimeoutCallback(
00199               PRFileDesc *sd, 
00200               PRFileDesc **nd,
00201               PRNetAddr **raddr, 
00202               void *buf, 
00203               PRInt32 amount, 
00204               PRIntervalTime t,
