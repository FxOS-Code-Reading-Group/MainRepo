00006 /* *************** SPS Sampler Information ****************
00007  *
00008  * SPS is an always on profiler that takes fast and low overheads samples
00009  * of the program execution using only userspace functionity for portability.
00010  * The goal of this module is to provide performance data in a generic
00011  * cross platform way without requiring custom tools or kernel support.
00012  *
00013  * Non goals: Support features that are platform specific or replace
00014  *            platform specific profilers.
00015  *
00016  * Samples are collected to form a timeline with optional timeline event (markers)
00017  * used for filtering.
00018  *
00019  * SPS collects samples in a platform independant way by using a speudo stack abstraction
00020  * of the real program stack by using 'sample stack frames'. When a sample is collected
00021  * all active sample stack frames and the program counter are recorded.
00022  */
00023 
00024 /* *************** SPS Sampler File Format ****************
00025  *
00026  * Simple new line seperated tag format:
00027  * S      -> BOF tags EOF
00028  * tags   -> tag tags
00029  * tag    -> CHAR - STRING
00030  *
00031  * Tags:
00032  * 's' - Sample tag followed by the first stack frame followed by 0 or more 'c' tags.
00033  * 'c' - Continue Sample tag gives remaining tag element. If a 'c' tag is seen without
00034  *         a preceding 's' tag it should be ignored. This is to support the behavior
00035  *         of circular buffers.
00036  *         If the 'stackwalk' feature is enabled this tag will have the format
00037  *         'l-<library name>@<hex address>' and will expect an external tool to translate
00038  *         the tag into something readable through a symbolication processing step.
00039  * 'm' - Timeline marker. Zero or more may appear before a 's' tag.
00040  * 'l' - Information about the program counter library and address. Post processing
00041  *         can include function and source line. If built with leaf data enabled
00042  *         this tag will describe the last 'c' tag.
00043  * 'r' - Responsiveness tag following an 's' tag. Gives an indication on how well the
00044  *          application is responding to the event loop. Lower is better.
00045  * 't' - Elapse time since recording started.
00046  *
00047  */
