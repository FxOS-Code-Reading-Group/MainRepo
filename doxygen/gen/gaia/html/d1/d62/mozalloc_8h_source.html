00064 /*
00065  * Each pair of declarations below is analogous to a "standard"
00066  * allocation function, except that the out-of-memory handling is made
00067  * explicit.  The |moz_x| versions will never return a NULL pointer;
00068  * if memory is exhausted, they abort.  The |moz_| versions may return
00069  * NULL pointers if memory is exhausted: their return value must be
00070  * checked.
00071  *
00072  * All these allocation functions are *guaranteed* to return a pointer
00073  * to memory allocated in such a way that that memory can be freed by
00074  * passing that pointer to |moz_free()|.

00155 /*
00156  * We implement the default operators new/delete as part of
00157  * libmozalloc, replacing their definitions in libstdc++.  The
00158  * operator new* definitions in libmozalloc will never return a NULL
00159  * pointer.
00160  *
00161  * Each operator new immediately below returns a pointer to memory
00162  * that can be delete'd by any of
00163  *
00164  *   (1) the matching infallible operator delete immediately below
00165  *   (2) the matching "fallible" operator delete further below
00166  *   (3) the matching system |operator delete(void*, std::nothrow)|
00167  *   (4) the matching system |operator delete(void*) throw(std::bad_alloc)|
00168  *
00169  * NB: these are declared |throw(std::bad_alloc)|, though they will never
00170  * throw that exception.  This declaration is consistent with the rule
00171  * that |::operator new() throw(std::bad_alloc)| will never return NULL.
00172  */
00173

00247 /*
00248  * We also add a new allocator variant: "fallible operator new."
00249  * Unlike libmozalloc's implementations of the standard nofail
00250  * allocators, this allocator is allowed to return NULL.  It can be used
00251  * as follows
00252  *
00253  *   Foo* f = new (mozilla::fallible) Foo(...);
