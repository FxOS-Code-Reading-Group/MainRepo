00251 /*
00252  *  Function to parse manifest file:
00253  *
00254  *  Many signatures may be attached to a single filename located
00255  *  inside the zip file. We only support one.
00256  *
00257  *  Several manifests may be included in the zip file.
00258  *
00259  *  You must pass the MANIFEST.MF file before any .SF files.
00260  *
00261  *  Right now this returns a big ole list, privately in the jar structure.
00262  *  If you need to traverse it, use JAR_find if possible.
00263  *
00264  *  The path is needed to determine what type of binary signature is
00265  *  being passed, though it is technically not needed for manifest files.
00266  *
00267  *  When parsing an ASCII file, null terminate the ASCII raw_manifest
00268  *  prior to sending it, and indicate a length of 0. For binary digital
00269  *  signatures only, indicate the true length of the signature.
00270  *  (This is legacy behavior.)
00271  *
00272  *  You may free the manifest after parsing it.
00273  *
00274  */
