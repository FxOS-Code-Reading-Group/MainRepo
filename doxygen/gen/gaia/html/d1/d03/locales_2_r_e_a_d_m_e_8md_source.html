00001 Building multilocale Gaia
00002 =========================
00003 
00004 You can build a multilocale Gaia profile with the following `make` command:
00005 
00006     make profile \
00007       LOCALE_BASEDIR=locales/ \
00008       LOCALES_FILE=locales/languages_basecamp.json \
      GAIA_DEFAULT_LOCALE=pt-BR
00009 
00010 Use `make multilocale-clean` to clean the working directory after the above.
00011 
00012 Use `GAIA_INLINE_LOCALES=1` to precompile all HTML to include text content in 
00013 the deafult locale.
00014 
00015 See `build/multilocale.py --help` for other l10n-related tasks that you may be 
00016 interested in (e.g., modifying the \*.ini files).
00017 
00018 The full documentation on building multilocale Gaia and B2G is avaiable on MDN: https://developer.mozilla.org/en-US/docs/Mozilla/Firefox_OS/Building#Building_multilocale

00021 Localization Testing
00022 ====================
00023 
00024 This directory can be used for localization testing.  You can put localization 
00025 files here and Gaia will use them when run in the `debug` mode.
00026 
00027 Running Gaia in debug mode
00028 --------------------------
00029 
00030 In order to enable the the `debug` mode (in a desktop build or in Firefox), 
00031 run:
00032 
00033     $ DEBUG=1 make profile
00034 
00035 Localization file layout
00036 ------------------------
00037 
00038 Files put in this directory should follow the l10n HG repos directory layout.  
00039 
00040 In the `debug` mode, Gaia will first try to find the `.properties` file in 
00041 
00042     locales/LOCALE/apps/APP/APP.properties
00043 
00044 For instance, the localization file for the browser app for French would be 
00045 located under:
00046 
00047     locales/fr/apps/browser/browser.properties
00048 
00049 Notice that this way `locales/fr` can be an HG clone of the French gaia-l10n 
00050 repository.
00051 
00052 If the file does not exist, Gaia will fallback to looking in its regular 
00053 location, i.e.:
00054 
00055     apps/APP/locales/APP.LOCALE.properties
00056 
00057 Use-case: working in HG
00058 -----------------------
00059 
00060 Clone your locale's Gaia l10n repository into this directory.  For instace, for 
00061 French (run from the git clone root):
00062 
00063     $ hg clone ssh://hg.mozilla.org/gaia-l10n/fr locales/fr
00064 
00065 Create Gaia's profile with:
00066 
00067     $ DEBUG=1 make profile
00068 
00069 And launch it in a desktop build:
00070 
00071     $ b2g-bin -profile profile-debug/
00072 
00073 ...or in Firefox:
00074 
00075     $ firefox -profile profile-debug/
00076 
00077 You can now use Gaia and test the localization.  When you make changes to the 
00078 files in the HG clone, just reload the Gaia app to see changes.
00079 
00080 Once you're done, commit your changes to HG from the clone you were working in.
