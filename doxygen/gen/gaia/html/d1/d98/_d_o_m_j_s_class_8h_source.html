00017 // We use slot 0 for holding the raw object.  This is safe for both
00018 // globals and non-globals.

00021 // We use slot 1 for holding the expando object. This is not safe for globals
00022 // until bug 760095 is fixed, so that bug blocks converting Window to new
00023 // bindings.
00024 #define DOM_XRAY_EXPANDO_SLOT 1
00025 
00026 // We use slot 2 for holding either a JS::ObjectValue which points to the cached
00027 // SOW or JS::UndefinedValue if this class doesn't need SOWs. This is not safe
00028 // for globals until bug 760095 is fixed, so that bug blocks converting Window
00029 // to new bindings.

00041 // changes.

00044 // Interface objects store a number of reserved slots equal to
00045 // DOM_INTERFACE_SLOTS_BASE + number of named constructors.
