virtual void mozilla::layers::CompositorOGL::MakeCurrent ( MakeCurrentFlags  aFlags = 0) [virtual]

Make this compositor's rendering context the current context for the underlying graphics API. This may be a global operation, depending on the API. Our context will remain the current one until someone else changes it.

Clients of the compositor should call this at the start of the compositing process, it might be required by texture uploads etc.

If aFlags == ForceMakeCurrent then we will (re-)set our context on the underlying API even if it is already the current context.

virtual void mozilla::layers::CompositorOGL::PrepareViewport ( const gfx::IntSize &  aSize,
const gfxMatrix &  aWorldTransform 
) [virtual]

Setup the viewport and projection matrix for rendering to a target of the given dimensions. The size and transform here will override those set in BeginFrame. BeginFrame sets a size and transform for the default render target, usually the screen. Calling this method prepares the compositor to render using a different viewport (that is, size and transform), usually associated with a new render target. aWorldTransform is the transform from user space to the new viewport's coordinate space.
