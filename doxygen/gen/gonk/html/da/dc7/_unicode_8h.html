void utf32_to_utf8 ( const char32_t *  src,
size_t  src_len,
char *  dst 
)

Stores a UTF-8 string converted from "src" in "dst", if "dst_length" is not large enough to store the string, the part of the "src" string is stored into "dst" as much as possible. See the examples for more detail. Returns the size actually used for storing the string. dst" is not null-terminated when dst_len is fully used (like strncpy).

Example 1 "src" == () "src_len" == 2 "dst_len" >= 7 -> Returned value == 6 "dst" becomes \0 (note that "dst" is null-terminated)

Example 2 "src" == () "src_len" == 2 "dst_len" == 5 -> Returned value == 3 "dst" becomes \0 (note that "dst" is null-terminated, but is not stored in "dst" since "dst" does not have enough size to store the character)

Example 3 "src" == () "src_len" == 2 "dst_len" == 6 -> Returned value == 6 "dst" becomes (note that "dst" is NOT null-terminated, like strncpy)

ssize_t utf8_length ( const char *  src)

Returns the length of "src" when "src" is valid UTF-8 string. Returns 0 if src is NULL or 0-length string. Returns -1 when the source is an invalid string.

This function should be used to determine whether "src" is valid UTF-8 characters with valid unicode codepoints. "src" must be null-terminated.

If you are going to use other utf8_to_... functions defined in this header with string which may not be valid UTF-8 with valid codepoint (form 0 to 0x10FFFF), you should use this function before calling others, since the other functions do not check whether the string is valid UTF-8 or not.

If you do not care whether "src" is valid UTF-8 or not, you should use strlen() as usual, which should be much faster.
